# DrawIO 功能部署说明

## 1. 数据库初始化

执行以下 SQL 脚本创建 DrawIO 图表表：

```bash
# 在 PostgreSQL 中执行
psql -U your_username -d your_database -f src/main/resources/sql/add_drawio_table.sql
```

或者直接在数据库管理工具中执行 `add_drawio_table.sql` 文件。

## 2. 配置问答模型

确保系统中已配置可用的问答模型：

1. 登录管理后台
2. 进入"大模型管理"
3. 配置至少一个问答模型（QAModel）
4. 确保模型已启用且可以正常调用

## 3. 常见问题排查

### 问题1: 500 Internal Server Error

**可能原因：**
- 数据库表未创建
- 未配置问答模型
- 模型调用失败

**解决方法：**
1. 检查后端日志，查看具体错误信息
2. 确认数据库表已创建：`SELECT * FROM DRAWIO_DIAGRAM LIMIT 1;`
3. 确认模型配置正确：检查 `QA_MODEL` 表是否有可用模型
4. 测试模型调用：在"智能问答"功能中测试模型是否正常工作

### 问题2: 模型调用失败

**可能原因：**
- 模型 API 地址配置错误
- API Key 无效
- 网络连接问题

**解决方法：**
1. 检查模型配置中的 API 地址和 Key
2. 在"大模型管理"中测试模型连接
3. 确保服务器可以访问模型 API

### 问题3: 图表生成失败

**可能原因：**
- LLM 返回的格式不正确
- XML 提取失败

**解决方法：**
1. 查看后端日志中的详细错误信息
2. 检查 LLM 返回的内容格式
3. 可能需要调整 prompt 或使用更强大的模型

## 4. 测试步骤

1. **测试数据库连接**
   ```sql
   SELECT COUNT(*) FROM DRAWIO_DIAGRAM;
   ```

2. **测试模型配置**
   - 在管理后台测试模型连接
   - 在"智能问答"中测试模型调用

3. **测试 DrawIO 功能**
   - 访问"AI 绘图"页面
   - 选择一个图表类型
   - 输入简单的描述（如："绘制一个简单的流程图，包含开始、处理、结束三个节点"）
   - 点击"生成图表"

## 5. 日志查看

查看后端日志文件：
- Windows: `logs/dify-app.log`
- Linux: `logs/dify-app.log`

搜索关键字：
- "生成图表请求"
- "生成图表失败"
- "获取问答模型失败"
