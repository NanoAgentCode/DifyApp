# 意图识别与备忘录确认设计文档

## 1. 背景

智能问答中存在一类用户输入并不是普通问答，而是希望系统执行写入类动作，例如“5 分钟后提醒我喝水”。这类请求不能完全交给大模型自由生成回复，否则模型可能在没有真实调用业务接口的情况下声称“已创建提醒”。

因此系统需要在调用大模型前先进行意图识别。当前第一阶段只支持备忘录意图，识别到完整提醒信息后只生成候选项，由前端弹出确认框，用户确认后才真正创建备忘录。

## 2. 设计目标

- 在模型生成内容前识别备忘录意图。
- 避免模型幻觉式回复“已创建提醒”。
- 写入类动作必须经过用户确认。
- 普通问答、知识库问答、文档问答等纯问答链路保持干净，不误触发业务写入。
- 将意图识别从 `ChatServiceImpl` 拆分为独立模块，便于后续扩展任务、日程、待办等意图。

## 3. 模块边界

当前调用链路：

```text
ChatController -> ChatService -> IntentRecognitionService -> MemoService
```

当前模块职责：

| 模块 | 职责 |
| --- | --- |
| `chat` | 接收普通聊天请求，在调用模型前调用意图识别服务 |
| `intent` | 识别备忘录意图，预览备忘录候选项，生成确认文案 |
| `memo` | 解析提醒时间与内容，用户确认后创建真实备忘录 |
| `frontend` | 展示确认框，确认后调用创建接口 |

## 4. 接入范围

| 功能链路 | 是否接入意图识别 | 说明 |
| --- | --- | --- |
| 普通智能问答 | 是 | 支持备忘录识别与确认创建 |
| 流式智能问答 | 是 | SSE 返回 `memoCandidate`，前端弹窗确认 |
| 用户端全局页面助手 | 默认否 | 页面助手默认关闭备忘录创建，只保留问答能力 |
| 知识库问答 | 否 | 只执行 RAG 检索和问答，不创建备忘录 |
| 文档问答 | 否 | 只基于文档上下文问答，不创建备忘录 |
| AI 应用 Chat Flow / Workflow | 否 | 由应用自身工作流定义动作能力 |

## 5. 后端设计

### 5.1 IntentRecognitionService

新增模块路径：

```text
backend/src/main/java/com/github/app/dify/intent/
```

核心接口：

```java
public interface IntentRecognitionService {
    boolean hasMemoIntent(String question);

    MemoResp previewMemo(Long userId, String question);

    String buildMemoConfirmationAnswer(MemoResp memo);
}
```

方法说明：

- `hasMemoIntent`：基于关键词判断用户是否表达备忘录或提醒意图。
- `previewMemo`：调用 `MemoService.preview` 解析提醒内容和提醒时间，但不写入数据库。
- `buildMemoConfirmationAnswer`：生成稳定确认文案，便于前端兜底解析。

### 5.2 ChatService 调用规则

普通聊天和流式聊天都在模型调用前执行以下逻辑：

1. 如果 `enableMemo=false`，跳过意图识别。
2. 如果 `previewMemo` 返回完整候选项，直接返回确认文案和 `memoCandidate`，不调用大模型。
3. 如果存在备忘录意图但提醒时间不完整，直接追问用户时间，不调用大模型。
4. 如果不是备忘录意图，继续正常聊天流程。

## 6. 前端设计

前端主聊天页面收到后端响应后执行：

1. 优先读取结构化字段 `memoCandidate`。
2. 弹出确认框，展示提醒内容和提醒时间。
3. 用户点击“创建”后调用 `POST /api/memos/confirmed`。
4. 用户取消或关闭弹窗时不创建。
5. 为兼容旧响应，如果 `memoCandidate` 缺失但出现固定确认文案，前端可以从文案解析候选项作为兜底。

## 7. API 设计

确认创建接口：

```text
POST /api/memos/confirmed
```

请求体：

```json
{
  "content": "喝水",
  "remindAt": "2026-05-24 13:08:06",
  "intervalMinutes": null
}
```

创建规则：

- 该接口只由用户确认动作触发。
- 聊天意图识别阶段不得直接调用创建接口。
- 预览阶段不得写入数据库。

## 8. 知识库问答边界

知识库问答不接入意图识别。

原因：

- 知识库问答的目标是基于知识库文档进行 RAG 检索和答案生成。
- 用户在知识库问答页输入的问题应被理解为针对知识库内容的查询。
- 知识库问答不应触发备忘录、待办、日程等写入类动作。
- 避免“查资料”场景误弹确认框，影响纯问答体验。

知识库问答链路保持为：

```text
KnowledgeBaseQAController -> KnowledgeBaseQAService -> RAG Retrieval -> LLM Answer
```

## 9. 失败与降级

- 意图识别失败：忽略失败，继续普通问答。
- 时间解析失败：如果有备忘录意图，追问用户具体提醒时间。
- 前端确认框取消：不创建备忘录。
- 创建接口失败：前端提示创建失败，聊天消息不声称已创建成功。

## 10. 后续扩展

后续可在 `intent` 模块中扩展更多意图类型：

- 待办任务
- 日程安排
- 系统设置修改
- 文件或知识库写入动作

扩展原则：

- 所有写入类动作都必须先生成候选项。
- 所有写入类动作都必须经过用户确认。
- 纯问答链路默认不接入写入类意图。
