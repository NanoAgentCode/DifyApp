# 错误码迁移指南

## 概述

本文档说明如何将系统中所有的异常替换为使用统一的错误码体系。已完成的改造包括：
- ✅ 全局异常处理器（消息脱敏）
- ✅ 错误码定义类（ErrorCode）
- ✅ BaseController 异常处理
- ✅ UnauthorizedException 支持错误码
- ✅ NotFoundException 支持错误码

## 迁移步骤

### 步骤 1: 理解错误码体系

使用 `ErrorCode` 类中预定义的错误码，而不是硬编码：

```java
// ❌ 错误做法
throw new RuntimeException("用户不存在");
throw new BusinessException("用户不存在", 1003);

// ✅ 正确做法
throw new NotFoundException("用户不存在");
// 或
throw new BusinessException("用户不存在", ErrorCode.USER_NOT_FOUND);
```

### 步骤 2: 异常类型映射

根据业务场景选择合适的异常类型：

| 异常类型 | 使用场景 | 默认错误码 |
|---------|---------|-----------|
| `NotFoundException` | 资源不存在 | 404 |
| `UnauthorizedException` | 未授权 | 401 |
| `BusinessException` | 业务逻辑错误 | 400 |
| `RuntimeException` | 通用异常 | 500 |

### 步骤 3: 常见错误码映射

#### 用户相关（1000-1999）

```java
// 用户不存在
throw new NotFoundException("用户不存在");
// 或
throw new BusinessException("用户不存在", ErrorCode.USER_NOT_FOUND);

// 登录失败
throw new BusinessException("用户名或密码错误", ErrorCode.LOGIN_FAILED);

// Token无效
throw new UnauthorizedException("Token无效");
// 或
throw new BusinessException("Token无效", ErrorCode.TOKEN_INVALID);

// 用户已存在
throw new BusinessException("用户已存在", ErrorCode.USER_ALREADY_EXISTS);
```

#### 资源相关（2000-2999）

```java
// 资源不存在
throw new NotFoundException("知识库不存在");
throw new NotFoundException("文档不存在");
throw new NotFoundException("应用不存在");
throw new NotFoundException("模型不存在");
```

#### 文件相关（3000-3999）

```java
// 文件上传失败
throw new BusinessException("文件上传失败", ErrorCode.FILE_UPLOAD_FAILED);

// 文件类型不支持
throw new BusinessException("不支持的文件类型: " + fileExtension, 
    ErrorCode.FILE_TYPE_NOT_SUPPORTED);

// 文件过大
throw new BusinessException("文件大小不能超过100MB", ErrorCode.FILE_TOO_LARGE);

// 文件不存在
throw new NotFoundException("文件不存在");
// 或
throw new BusinessException("文件不存在", ErrorCode.FILE_NOT_FOUND);
```
