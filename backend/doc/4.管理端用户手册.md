## 系统简介

DifyApp 管理端是系统管理员专用的管理界面，提供完整的系统管理功能，包括：

- **应用管理**：创建、编辑、删除 AI 应用，配置应用参数
- **用户管理**：用户审核、权限管理、角色分配
- **知识库管理**：创建和管理知识库，管理文档
- **大模型管理**：配置问答模型和向量化模型（支持视觉模型配置）
- **向量数据库管理**：配置和管理向量数据库
- **对话历史管理**：查看所有用户的对话记录
- **AI 绘图**：使用自然语言生成各种类型的图表
- **数据源管理**：配置和管理数据库连接（支持 PostgreSQL、MySQL、Oracle、MongoDB、Neo4j、Elasticsearch 等）
- **系统配置管理**：管理系统级别的配置参数
- **数据分析**：Neo4j 图数据库数据同步和可视化
- **用户行为日志**：用户操作日志查询和分析（基于 Elasticsearch）
- **系统监控**：查看系统使用统计
- **OCR服务配置**：配置OCR服务，支持图片和PDF文字识别

## 快速开始

### 1. 登录系统

1. 打开系统登录页面
2. 使用管理员账号登录
3. 登录成功后，系统会自动跳转到管理端首页

> **提示**：默认管理员账号为 `admin`，首次登录后请立即修改密码。

### 2. 管理端界面

管理端界面包含以下主要区域：

- **顶部导航栏**：显示系统标题和用户信息
- **左侧菜单**：功能模块导航
- **主内容区**：显示当前功能页面

## 功能模块

### 3.1 应用管理

应用管理功能允许您创建、编辑、删除和查看 AI 应用。

#### 3.1.1 创建应用

1. 在左侧菜单中点击"应用列表"
2. 点击"创建应用"按钮
3. 填写应用信息：
   - **应用名称**（必填）：应用的显示名称
   - **应用描述**（可选）：应用的简要描述
   - **应用类型**（必填）：
     - **Chat Flow**：对话式应用
     - **Workflow**：工作流应用
   - **Dify 应用 ID**（必填）：Dify 平台中的应用 ID
   - **API Base URL**（必填）：Dify API 的基础地址
   - **API Key**（必填）：Dify 应用的 API Key
   - **租户 ID**（可选）：多租户场景下的租户标识
   - **流式响应**：是否启用流式响应
   - **文件上传**：是否支持文件上传
   - **输入启用**：是否启用输入功能
   - **主题颜色**（可选）：应用的主题颜色
   - **输入配置**（可选）：JSON 格式的输入配置
4. 点击"确定"创建应用

#### 3.1.2 查看应用列表

应用列表以表格形式展示，包含以下信息：

- 应用 ID
- 应用图标
- 应用名称
- 应用描述
- 应用类型（Chat Flow / Workflow）
- 应用状态（启用/禁用）
- 是否支持流式响应
- 创建时间

**搜索和筛选**：
- **搜索**：按应用名称或描述搜索
- **类型筛选**：按应用类型筛选
- **状态筛选**：按应用状态筛选

#### 3.1.3 编辑应用

1. 在应用列表中，找到要编辑的应用
2. 点击"编辑"按钮
3. 修改应用信息
4. 点击"确定"保存修改

#### 3.1.4 查看应用详情

1. 在应用列表中，找到要查看的应用
2. 点击"详情"按钮
3. 查看应用的完整信息，包括：
   - 基本信息
   - API 配置
   - 功能开关
   - 使用统计

#### 3.1.5 使用应用

1. 在应用列表中，找到要使用的应用
2. 点击"使用"按钮
3. 系统会根据应用类型跳转到相应的交互界面

#### 3.1.6 删除应用

1. 在应用列表中，找到要删除的应用
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除应用将同时删除所有关联的对话历史，此操作不可恢复！

### 3.2 用户管理

用户管理功能允许您管理所有系统用户，包括用户审核、权限管理、角色分配等。

#### 3.2.1 查看用户列表

用户列表以表格形式展示，包含以下信息：

- 用户 ID
- 用户名
- 角色（管理员/普通用户）
- 状态（待审核/已激活/已禁用）
- 邮箱
- 创建时间

**搜索和筛选**：
- **搜索**：按用户名搜索
- **状态筛选**：按用户状态筛选
- **角色筛选**：按用户角色筛选

#### 3.2.2 审核用户

新用户注册后，状态为"待审核"，需要管理员审核后才能使用系统。

1. 在用户列表中，找到状态为"待审核"的用户
2. 点击"审核"按钮
3. 确认审核操作

审核通过后，用户状态变为"已激活"，可以正常使用系统。

#### 3.2.3 禁用/启用用户

1. 在用户列表中，找到要操作的用户
2. 点击"禁用"或"启用"按钮
3. 确认操作

**禁用用户**：
- 禁用后，用户无法登录系统
- 已禁用的用户不会显示在用户列表中（除非筛选显示）

**启用用户**：
- 启用后，用户可以正常使用系统

#### 3.2.4 修改用户角色

1. 在用户列表中，找到要修改角色的用户
2. 在"角色"列中，点击下拉框
3. 选择新的角色：
   - **管理员**：拥有所有权限
   - **普通用户**：只能访问被授权的资源
4. 系统会自动保存修改

> **注意**：不能修改超级管理员（默认 admin 用户）的角色。

#### 3.2.5 重置用户密码

1. 在用户列表中，找到要重置密码的用户
2. 点击"重置密码"按钮
3. 输入新密码
4. 确认新密码
5. 点击"确定"完成重置

> **提示**：重置密码后，请通知用户使用新密码登录，并建议用户首次登录后修改密码。

#### 3.2.6 应用权限管理

您可以为每个用户设置应用可见性，控制用户可以访问哪些应用。

1. 在用户列表中，找到要管理的用户
2. 点击"应用管理"按钮
3. 在应用权限管理界面中：
   - 查看所有应用列表
   - 切换应用的可见性（可见/不可见）
   - 查看可见应用数量统计

**权限说明**：
- **管理员**：默认拥有所有应用的访问权限，不可修改
- **普通用户**：只能访问被设置为可见的应用

#### 3.2.7 知识库权限管理

您可以为每个用户设置知识库可见性，控制用户可以访问哪些知识库。

1. 在用户列表中，找到要管理的用户
2. 点击"知识库管理"按钮
3. 在知识库权限管理界面中：
   - 查看所有知识库列表
   - 切换知识库的可见性（可见/不可见）
   - 查看可见知识库数量统计

**权限说明**：
- **管理员**：默认拥有所有知识库的访问权限，不可修改
- **普通用户**：只能访问被设置为可见的知识库

#### 3.2.8 数据源权限管理

您可以为每个用户设置数据源可见性，控制用户可以访问哪些数据源。

1. 在用户列表中，找到要管理的用户
2. 点击"数据源管理"按钮
3. 在数据源权限管理界面中：
   - 查看所有数据源列表
   - 切换数据源的可见性（可见/不可见）
   - 批量设置数据源可见性
   - 查看可见数据源数量统计

**权限说明**：
- **管理员**：默认拥有所有数据源的访问权限，不可修改
- **普通用户**：只能访问被设置为可见的数据源

### 3.3 知识库管理

知识库管理功能允许您创建、编辑、删除知识库，管理文档等。

#### 3.3.1 创建知识库

1. 在左侧菜单中点击"知识库管理"
2. 点击"创建知识库"按钮
3. 填写知识库信息：
   - **知识库名称**（必填）：知识库的显示名称
   - **描述**（可选）：知识库的简要描述
   - **向量化模型**（可选）：选择用于文档向量化的模型
   - **向量存储类型**（必填）：选择向量存储方式
     - **Qdrant**：分布式向量数据库（默认）
     - **FAISS**：本地文件存储
     - **Milvus**：分布式向量数据库
     - **Chroma**：轻量级向量数据库
     - **Weaviate**：开源向量数据库
     - **PgVector**：PostgreSQL扩展
     - **Elasticsearch**：分布式搜索和分析引擎
4. 点击"确定"创建知识库

> **注意**：知识库创建后，向量存储类型无法修改。

#### 3.3.2 查看知识库列表

知识库列表以表格形式展示，包含以下信息：

- 知识库 ID
- 知识库名称
- 描述
- 文档数量
- 可见性状态（公开/私有）
- 状态（启用/禁用）
- 向量化模型
- 向量存储类型
- 创建时间

**搜索和筛选**：
- **搜索**：按知识库名称或描述搜索
- **状态筛选**：按知识库状态筛选

#### 3.3.3 编辑知识库

1. 在知识库列表中，找到要编辑的知识库
2. 点击"编辑"按钮
3. 修改知识库信息（注意：向量存储类型无法修改）
4. 点击"确定"保存修改

#### 3.3.4 删除知识库

1. 在知识库列表中，找到要删除的知识库
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除知识库将同时删除所有关联的文档和向量数据，此操作不可恢复！

#### 3.3.5 文档管理

1. 在知识库列表中，找到要管理文档的知识库
2. 点击"文档管理"按钮
3. 进入文档管理页面

**文档管理功能**：
- **上传文档**：上传新文档到知识库
- **查看文档列表**：查看知识库中的所有文档
- **下载文档**：下载原始文档文件
- **删除文档**：删除不需要的文档
- **重新向量化**：重新进行文档向量化处理

#### 3.3.6 上传文档

1. 在文档管理页面，点击"上传文档"按钮
2. 选择要上传的文件（支持 PDF、Word、TXT、Markdown 等格式）
3. 点击"确定"开始上传

**文档处理流程**：
1. 文档上传到 MinIO 对象存储
2. 根据文档类型选择处理方式：
   - **图片文件和PDF文件**：优先使用OCR服务识别文字
   - **Word文档**：提取图片进行OCR识别，与文本内容合并
   - **其他格式**：使用Apache Tika解析文档内容
3. 文档内容分块（可配置分块大小和重叠）
4. 使用向量化模型将文档块转换为向量
5. 向量数据存储到向量数据库

**OCR识别说明**：
- 系统集成了EasyOCR服务，支持图片和PDF文字识别
- Word文档中的图片也会进行OCR识别
- 如果OCR服务不可用，系统会自动回退到文本解析

**文档状态**：
- **pending**：等待处理
- **processing**：正在处理中
- **completed**：处理完成
- **failed**：处理失败

#### 3.3.7 文档格式支持

系统支持以下文档格式：

- PDF（.pdf）- 支持OCR文字识别
- Word（.doc, .docx）- 支持文本提取和图片OCR识别
- 文本文件（.txt）
- Markdown（.md）
- 图片文件（.png, .jpg, .jpeg, .gif, .bmp等）- 支持OCR文字识别
- Excel（.xls, .xlsx）
- PowerPoint（.ppt, .pptx）
- HTML（.html）
- 其他格式（根据系统配置）

**OCR识别说明**：
- 图片文件和PDF文件会自动使用OCR服务识别文字
- Word文档中的图片也会进行OCR识别，并与文本内容合并
- 如果OCR服务不可用，系统会自动回退到文本解析

### 3.4 知识库问答

知识库问答功能允许您基于知识库内容进行智能问答，用于测试知识库效果。

#### 3.4.1 开始问答

1. 在左侧菜单中点击"知识库问答"
2. 选择要使用的知识库
3. 选择问答模型（可选，不选择则使用默认模型）
4. 在输入框中输入您的问题
5. 点击"发送"按钮或按 Enter 键发送

#### 3.4.2 问答功能

- **流式响应**：答案会实时显示
- **非流式响应**：一次性返回完整答案
- **连续对话**：支持多轮对话
- **答案来源**：显示答案引用的文档片段

#### 3.4.3 模型选择

- 系统会显示可用的问答模型列表
- 选择模型后，系统会使用该模型进行问答

### 3.5 对话历史管理

对话历史管理功能允许您查看所有用户的对话记录，进行系统监控和分析。

#### 3.5.1 查看对话历史

1. 在左侧菜单中点击"会话历史"
2. 系统会显示所有用户的对话会话列表
3. 会话列表以表格形式展示，包含：
   - 会话 ID
   - 用户信息
   - 应用/知识库信息
   - 会话类型（普通聊天/知识库问答）
   - 会话标题
   - 消息数量
   - 最后消息时间
   - 创建时间

#### 3.5.2 搜索和筛选

- **搜索**：按会话标题或内容搜索
- **用户筛选**：按用户筛选
- **类型筛选**：按会话类型筛选（普通聊天/知识库问答）
- **分页**：支持分页浏览

#### 3.5.3 查看会话详情

1. 在会话列表中，找到要查看的会话
2. 点击"查看"按钮
3. 系统会显示该会话的所有消息
4. 消息以对话形式展示，支持 Markdown 渲染、代码高亮、数学公式

#### 3.5.4 更新会话标题

1. 在会话详情页面，点击"编辑标题"按钮
2. 输入新的会话标题
3. 点击"确定"保存修改

#### 3.5.5 导出会话

1. 在会话详情页面，点击"导出"按钮
2. 系统会生成包含所有消息的导出文件
3. 您可以保存或分享导出的会话内容

#### 3.5.6 删除会话

1. 在会话列表中，找到要删除的会话
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除会话将同时删除所有关联的消息，此操作不可恢复！

#### 3.5.7 对话统计

系统提供对话历史统计功能，包括：

- **总会话数**：系统中的总对话会话数量
- **总消息数**：系统中的总消息数量
- **今日会话数**：今日创建的会话数量
- **今日消息数**：今日发送的消息数量

### 3.6 大模型管理

大模型管理功能允许您配置和管理问答模型和向量化模型。

#### 3.6.1 问答模型管理

问答模型用于智能问答和知识库问答功能。

**查看模型列表**：
1. 在左侧菜单中点击"大模型管理"
2. 切换到"问答模型"标签页
3. 查看所有问答模型列表

**模型信息**：
- 模型名称
- 模型标识（带颜色区分）
- 提供商类型（OpenAI、Ollama、VLLM 等）
- API 地址
- 使用场景（智能问答/知识库问答/两者都支持）
- 支持多模态（是否支持多模态输入）
- 支持视觉输入（是否支持视觉输入）
- 是否启用
- 是否默认模型

**添加模型**：
1. 点击"添加模型"按钮
2. 填写模型信息：
   - **模型名称**（必填）：模型的显示名称
   - **提供商类型**（必填）：选择提供商（OpenAI、Ollama、VLLM 等）
   - **API 地址**（必填）：模型的 API 地址
   - **API Key**（可选）：API 密钥
   - **模型标识**（必填）：模型的标识符
   - **使用场景**（必填）：选择使用场景
     - **智能问答**：仅用于智能问答
     - **知识库问答**：仅用于知识库问答
     - **两者都支持**：两种场景都支持
   - **支持多模态**：是否支持多模态输入（文本+图片）
   - **支持视觉输入**：是否支持视觉输入（图片理解、文字识别等）
   - **是否启用**：是否启用该模型
3. 点击"确定"添加模型

> **提示**：系统会根据模型名称自动检测是否支持视觉输入（如包含"vision"、"vl"等关键词）。您也可以手动设置这两个选项。

**编辑模型**：
1. 在模型列表中，找到要编辑的模型
2. 点击"编辑"按钮
3. 修改模型信息
4. 点击"确定"保存修改

**删除模型**：
1. 在模型列表中，找到要删除的模型
2. 点击"删除"按钮
3. 确认删除操作

> **注意**：不能删除正在使用的模型，删除前请确保没有应用或知识库使用该模型。

**设置默认模型**：

### 3.10 AI 绘图

AI 绘图功能允许您使用自然语言生成各种类型的图表，包括流程图、架构图、思维导图、时序图等。

#### 3.10.1 开始绘图

1. 在左侧菜单中点击"AI 绘图"
2. 选择图表类型（可选，不选择则使用默认类型）
3. 在输入框中输入您的绘图需求（自然语言描述）
4. 点击"生成"按钮或按 Enter 键

#### 3.10.2 支持的图表类型

系统支持以下图表类型：

- **流程图（flowchart）**：用于描述业务流程、决策流程等
- **架构图（architecture）**：用于描述系统架构、技术架构等
- **思维导图（mindmap）**：用于组织思维、知识结构等
- **时序图（sequence）**：用于描述系统交互序列
- **UML 图（uml）**：用于描述类图、用例图等
- **组织架构图（org）**：用于描述组织架构
- **网络图（network）**：用于描述网络拓扑
- **自定义图表（custom）**：根据需求自定义图表类型

#### 3.10.3 图表生成

**生成步骤**：
1. 输入自然语言描述，例如："绘制用户登录流程图，包含输入账号密码、验证、登录成功、登录失败等步骤"
2. 选择图表类型（可选）
3. 选择模型（可选，不选择则使用系统默认模型）
4. 点击"生成"按钮
5. 系统会使用 AI 生成 Mermaid 格式的图表代码并自动渲染

**生成结果**：
- 图表会实时渲染在画布上
- 支持缩放、拖拽等交互操作
- 可以查看生成的 Mermaid 代码

#### 3.10.4 图表修改

1. 在画布上已有图表的情况下，在输入框中输入修改指令
2. 例如："添加忘记密码功能"
3. 点击"修改"按钮
4. 系统会根据修改指令更新图表

#### 3.10.5 图表保存

2. 输入图表名称
3. 点击"确定"保存图表

### 3.13 数据分析

数据分析模块提供基于 Neo4j 图数据库的数据分析和可视化功能，可以将系统数据同步到图数据库进行关系分析。

#### 3.13.1 功能概述

- **数据同步**：将用户、应用、知识库、文档、对话、消息等实体同步到 Neo4j 图数据库
- **关系构建**：自动构建实体之间的关系（用户-应用、用户-知识库、对话-应用等）
- **图数据可视化**：提供图数据查询和可视化展示（节点和关系）
- **定时同步**：支持定时自动同步（可配置同步间隔）
- **同步状态监控**：记录同步时间、状态、指标等信息

#### 3.13.2 配置 Neo4j 数据源

在使用数据分析功能之前，需要先配置 Neo4j 数据源：

1. 进入「数据源管理」页面
2. 点击「创建数据源」按钮
3. 填写数据源信息：
   - **数据源名称**：例如 "Neo4j 图数据库"
   - **数据库类型**：选择 `Neo4j`
   - **主机地址**：Neo4j 服务器地址（如 `localhost`）
   - **端口**：Neo4j Bolt 端口（默认 `7687`）
   - **数据库名称**：可填写任意非空字符串（如 `neo4j`）
   - **用户名**：Neo4j 用户名（如 `neo4j`）
   - **密码**：Neo4j 密码
   - **状态**：启用
4. 点击「测试连接」验证连接是否成功
5. 点击「保存」创建数据源
6. 记下该数据源的 **ID**

#### 3.13.3 配置同步设置

1. 进入「数据分析」页面
2. 点击「同步设置」按钮
3. 在设置对话框中：
   - **Neo4j 数据源**：选择上面创建的 Neo4j 数据源
   - **定时同步**：开启/关闭定时同步
   - **同步间隔**：设置同步间隔（分钟，范围 1-1440，默认 60 分钟）
4. 点击「保存」保存设置

#### 3.13.4 执行数据同步

**立即同步**：
1. 在「数据分析」页面，点击「立即同步」按钮
2. 系统会立即执行数据同步任务
3. 同步过程中会显示同步状态

**定时同步**：
- 如果开启了定时同步，系统会按照设置的间隔自动执行同步任务
- 首次同步会在设置保存后立即执行

#### 3.13.5 查看同步状态

在「数据分析」页面可以查看：
- **同步状态**：最近一次同步的状态（成功/失败/运行中/未启用）
- **同步时间**：最近一次同步的开始时间和成功时间
- **同步耗时**：最近一次同步的执行时间
- **同步指标**：同步的数据量（用户数、应用数、知识库数等）

#### 3.13.6 查看图数据可视化

1. 在「数据分析」页面，系统会自动加载图数据
2. 图数据以交互式图表形式展示：
   - **节点**：表示实体（用户、应用、知识库等）
   - **关系**：表示实体之间的关系
   - **节点统计**：显示各类型节点的数量
   - **关系统计**：显示各类型关系的数量
3. 可以通过鼠标拖拽、缩放等方式查看图数据

#### 3.13.7 注意事项

- Neo4j 数据源必须正确配置且处于启用状态
- 首次同步可能需要较长时间，取决于数据量
- 同步过程中不会影响系统的正常使用
- 建议在系统低峰期执行同步任务

### 3.14 用户行为日志

#### 3.14.1 功能概述

- **日志分析**：操作类型统计、模块统计等聚合分析
- **高性能检索**：基于 Elasticsearch 实现毫秒级检索
- **多维度筛选**：支持按用户、模块、操作类型、时间范围等条件筛选

   - **主机地址**：Elasticsearch 服务器地址（如 `localhost`）

   - **数据库名称**：可填写任意非空字符串（内部仅用于区分连接）
   - **用户名**：如果 Elasticsearch 开启了安全认证，则填写对应账号；否则可留空

**A**:

2. 检查向量化模型是否正常
3. 检查向量数据库连接是否正常

**OCR相关问题**：

- OCR 请求超时时间统一为 30 秒（固定值，不再通过系统配置项设置）

### Q21: 向量数据库支持哪些类型？

**A**: 系统支持以下向量数据库：
- **Qdrant**：分布式向量数据库，适合生产环境
- **Milvus**：分布式向量数据库，适合生产环境
- **FAISS**：本地文件存储，适合开发测试环境
- **Chroma**：轻量级向量数据库，适合中小规模应用
- **Weaviate**：开源向量数据库，支持多种向量化模型
- **PgVector**：PostgreSQL扩展，适合已有PostgreSQL环境的场景
- **Elasticsearch**：分布式搜索和分析引擎，支持向量搜索

### Q22: 如何选择向量数据库？

**A**: 选择建议：
- **开发测试环境**：使用 FAISS（无需额外服务）
- **中小规模应用**：使用 Chroma
- **生产环境**：使用 Qdrant、Milvus 或 Weaviate
- **已有PostgreSQL环境**：使用 PgVector（无需额外服务）
- **需要搜索功能**：使用 Elasticsearch
