## 系统简介

DifyApp 管理端是系统管理员专用的管理界面，提供完整的系统管理功能，包括：

- **应用管理**：创建、编辑、删除 AI 应用，配置应用参数
- **用户管理**：用户审核、权限管理、角色分配
- **知识库管理**：创建和管理知识库，管理文档
- **大模型管理**：配置问答模型和向量化模型
- **向量数据库管理**：配置和管理向量数据库
- **对话历史管理**：查看所有用户的对话记录
- **Text2SQL**：自然语言转 SQL 查询功能
- **AI 绘图**：使用自然语言生成各种类型的图表
- **数据源管理**：配置和管理数据库连接
- **系统配置管理**：管理系统级别的配置参数
- **系统监控**：查看系统使用统计

## 快速开始

### 1. 登录系统

1. 打开系统登录页面
2. 使用管理员账号登录
3. 登录成功后，系统会自动跳转到管理端首页

> **提示**：默认管理员账号为 `admin`，首次登录后请立即修改密码。

### 2. 管理端界面

管理端界面包含以下主要区域：

- **顶部导航栏**：显示系统标题和用户信息
- **左侧菜单**：功能模块导航
- **主内容区**：显示当前功能页面

## 功能模块

### 3.1 应用管理

应用管理功能允许您创建、编辑、删除和查看 AI 应用。

#### 3.1.1 创建应用

1. 在左侧菜单中点击"应用列表"
2. 点击"创建应用"按钮
3. 填写应用信息：
   - **应用名称**（必填）：应用的显示名称
   - **应用描述**（可选）：应用的简要描述
   - **应用类型**（必填）：
     - **Chat Flow**：对话式应用
     - **Workflow**：工作流应用
   - **Dify 应用 ID**（必填）：Dify 平台中的应用 ID
   - **API Base URL**（必填）：Dify API 的基础地址
   - **API Key**（必填）：Dify 应用的 API Key
   - **租户 ID**（可选）：多租户场景下的租户标识
   - **流式响应**：是否启用流式响应
   - **文件上传**：是否支持文件上传
   - **输入启用**：是否启用输入功能
   - **主题颜色**（可选）：应用的主题颜色
   - **输入配置**（可选）：JSON 格式的输入配置
4. 点击"确定"创建应用

#### 3.1.2 查看应用列表

应用列表以表格形式展示，包含以下信息：

- 应用 ID
- 应用图标
- 应用名称
- 应用描述
- 应用类型（Chat Flow / Workflow）
- 应用状态（启用/禁用）
- 是否支持流式响应
- 创建时间

**搜索和筛选**：
- **搜索**：按应用名称或描述搜索
- **类型筛选**：按应用类型筛选
- **状态筛选**：按应用状态筛选

#### 3.1.3 编辑应用

1. 在应用列表中，找到要编辑的应用
2. 点击"编辑"按钮
3. 修改应用信息
4. 点击"确定"保存修改

#### 3.1.4 查看应用详情

1. 在应用列表中，找到要查看的应用
2. 点击"详情"按钮
3. 查看应用的完整信息，包括：
   - 基本信息
   - API 配置
   - 功能开关
   - 使用统计

#### 3.1.5 使用应用

1. 在应用列表中，找到要使用的应用
2. 点击"使用"按钮
3. 系统会根据应用类型跳转到相应的交互界面

#### 3.1.6 删除应用

1. 在应用列表中，找到要删除的应用
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除应用将同时删除所有关联的对话历史，此操作不可恢复！

### 3.2 用户管理

用户管理功能允许您管理所有系统用户，包括用户审核、权限管理、角色分配等。

#### 3.2.1 查看用户列表

用户列表以表格形式展示，包含以下信息：

- 用户 ID
- 用户名
- 角色（管理员/普通用户）
- 状态（待审核/已激活/已禁用）
- 邮箱
- 创建时间

**搜索和筛选**：
- **搜索**：按用户名搜索
- **状态筛选**：按用户状态筛选
- **角色筛选**：按用户角色筛选

#### 3.2.2 审核用户

新用户注册后，状态为"待审核"，需要管理员审核后才能使用系统。

1. 在用户列表中，找到状态为"待审核"的用户
2. 点击"审核"按钮
3. 确认审核操作

审核通过后，用户状态变为"已激活"，可以正常使用系统。

#### 3.2.3 禁用/启用用户

1. 在用户列表中，找到要操作的用户
2. 点击"禁用"或"启用"按钮
3. 确认操作

**禁用用户**：
- 禁用后，用户无法登录系统
- 已禁用的用户不会显示在用户列表中（除非筛选显示）

**启用用户**：
- 启用后，用户可以正常使用系统

#### 3.2.4 修改用户角色

1. 在用户列表中，找到要修改角色的用户
2. 在"角色"列中，点击下拉框
3. 选择新的角色：
   - **管理员**：拥有所有权限
   - **普通用户**：只能访问被授权的资源
4. 系统会自动保存修改

> **注意**：不能修改超级管理员（默认 admin 用户）的角色。

#### 3.2.5 重置用户密码

1. 在用户列表中，找到要重置密码的用户
2. 点击"重置密码"按钮
3. 输入新密码
4. 确认新密码
5. 点击"确定"完成重置

> **提示**：重置密码后，请通知用户使用新密码登录，并建议用户首次登录后修改密码。

#### 3.2.6 应用权限管理

您可以为每个用户设置应用可见性，控制用户可以访问哪些应用。

1. 在用户列表中，找到要管理的用户
2. 点击"应用管理"按钮
3. 在应用权限管理界面中：
   - 查看所有应用列表
   - 切换应用的可见性（可见/不可见）
   - 查看可见应用数量统计

**权限说明**：
- **管理员**：默认拥有所有应用的访问权限，不可修改
- **普通用户**：只能访问被设置为可见的应用

#### 3.2.7 知识库权限管理

您可以为每个用户设置知识库可见性，控制用户可以访问哪些知识库。

1. 在用户列表中，找到要管理的用户
2. 点击"知识库管理"按钮
3. 在知识库权限管理界面中：
   - 查看所有知识库列表
   - 切换知识库的可见性（可见/不可见）
   - 查看可见知识库数量统计

**权限说明**：
- **管理员**：默认拥有所有知识库的访问权限，不可修改
- **普通用户**：只能访问被设置为可见的知识库

### 3.3 知识库管理

知识库管理功能允许您创建、编辑、删除知识库，管理文档等。

#### 3.3.1 创建知识库

1. 在左侧菜单中点击"知识库管理"
2. 点击"创建知识库"按钮
3. 填写知识库信息：
   - **知识库名称**（必填）：知识库的显示名称
   - **描述**（可选）：知识库的简要描述
   - **向量化模型**（可选）：选择用于文档向量化的模型
   - **向量存储类型**（必填）：选择向量存储方式
     - **Qdrant**：分布式向量数据库（默认）
     - **FAISS**：本地文件存储
     - **Milvus**：分布式向量数据库
4. 点击"确定"创建知识库

> **注意**：知识库创建后，向量存储类型无法修改。

#### 3.3.2 查看知识库列表

知识库列表以表格形式展示，包含以下信息：

- 知识库 ID
- 知识库名称
- 描述
- 文档数量
- 可见性状态（公开/私有）
- 状态（启用/禁用）
- 向量化模型
- 向量存储类型
- 创建时间

**搜索和筛选**：
- **搜索**：按知识库名称或描述搜索
- **状态筛选**：按知识库状态筛选

#### 3.3.3 编辑知识库

1. 在知识库列表中，找到要编辑的知识库
2. 点击"编辑"按钮
3. 修改知识库信息（注意：向量存储类型无法修改）
4. 点击"确定"保存修改

#### 3.3.4 删除知识库

1. 在知识库列表中，找到要删除的知识库
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除知识库将同时删除所有关联的文档和向量数据，此操作不可恢复！

#### 3.3.5 文档管理

1. 在知识库列表中，找到要管理文档的知识库
2. 点击"文档管理"按钮
3. 进入文档管理页面

**文档管理功能**：
- **上传文档**：上传新文档到知识库
- **查看文档列表**：查看知识库中的所有文档
- **下载文档**：下载原始文档文件
- **删除文档**：删除不需要的文档
- **重新向量化**：重新进行文档向量化处理

#### 3.3.6 上传文档

1. 在文档管理页面，点击"上传文档"按钮
2. 选择要上传的文件（支持 PDF、Word、TXT、Markdown 等格式）
3. 点击"确定"开始上传

**文档处理流程**：
1. 文档上传到 MinIO 对象存储
2. 使用 Apache Tika 解析文档内容
3. 文档内容分块（可配置分块大小和重叠）
4. 使用向量化模型将文档块转换为向量
5. 向量数据存储到向量数据库

**文档状态**：
- **pending**：等待处理
- **processing**：正在处理中
- **completed**：处理完成
- **failed**：处理失败

#### 3.3.7 文档格式支持

系统支持以下文档格式：

- PDF（.pdf）
- Word（.doc, .docx）
- 文本文件（.txt）
- Markdown（.md）
- 其他格式（根据系统配置）

### 3.4 知识库问答

知识库问答功能允许您基于知识库内容进行智能问答，用于测试知识库效果。

#### 3.4.1 开始问答

1. 在左侧菜单中点击"知识库问答"
2. 选择要使用的知识库
3. 选择问答模型（可选，不选择则使用默认模型）
4. 在输入框中输入您的问题
5. 点击"发送"按钮或按 Enter 键发送

#### 3.4.2 问答功能

- **流式响应**：答案会实时显示
- **非流式响应**：一次性返回完整答案
- **连续对话**：支持多轮对话
- **答案来源**：显示答案引用的文档片段

#### 3.4.3 模型选择

- 系统会显示可用的问答模型列表
- 选择模型后，系统会使用该模型进行问答

### 3.5 对话历史管理

对话历史管理功能允许您查看所有用户的对话记录，进行系统监控和分析。

#### 3.5.1 查看对话历史

1. 在左侧菜单中点击"会话历史"
2. 系统会显示所有用户的对话会话列表
3. 会话列表以表格形式展示，包含：
   - 会话 ID
   - 用户信息
   - 应用/知识库信息
   - 会话类型（普通聊天/知识库问答）
   - 会话标题
   - 消息数量
   - 最后消息时间
   - 创建时间

#### 3.5.2 搜索和筛选

- **搜索**：按会话标题或内容搜索
- **用户筛选**：按用户筛选
- **类型筛选**：按会话类型筛选（普通聊天/知识库问答）
- **分页**：支持分页浏览

#### 3.5.3 查看会话详情

1. 在会话列表中，找到要查看的会话
2. 点击"查看"按钮
3. 系统会显示该会话的所有消息
4. 消息以对话形式展示，支持 Markdown 渲染、代码高亮、数学公式

#### 3.5.4 更新会话标题

1. 在会话详情页面，点击"编辑标题"按钮
2. 输入新的会话标题
3. 点击"确定"保存修改

#### 3.5.5 导出会话

1. 在会话详情页面，点击"导出"按钮
2. 系统会生成包含所有消息的导出文件
3. 您可以保存或分享导出的会话内容

#### 3.5.6 删除会话

1. 在会话列表中，找到要删除的会话
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除会话将同时删除所有关联的消息，此操作不可恢复！

#### 3.5.7 对话统计

系统提供对话历史统计功能，包括：

- **总会话数**：系统中的总对话会话数量
- **总消息数**：系统中的总消息数量
- **今日会话数**：今日创建的会话数量
- **今日消息数**：今日发送的消息数量

### 3.6 大模型管理

大模型管理功能允许您配置和管理问答模型和向量化模型。

#### 3.6.1 问答模型管理

问答模型用于智能问答和知识库问答功能。

**查看模型列表**：
1. 在左侧菜单中点击"大模型管理"
2. 切换到"问答模型"标签页
3. 查看所有问答模型列表

**模型信息**：
- 模型名称
- 模型标识（带颜色区分）
- 提供商类型（OpenAI、Ollama、VLLM 等）
- API 地址
- 使用场景（智能问答/知识库问答/两者都支持）
- 是否启用
- 是否默认模型

**添加模型**：
1. 点击"添加模型"按钮
2. 填写模型信息：
   - **模型名称**（必填）：模型的显示名称
   - **提供商类型**（必填）：选择提供商（OpenAI、Ollama、VLLM 等）
   - **API 地址**（必填）：模型的 API 地址
   - **API Key**（可选）：API 密钥
   - **模型标识**（必填）：模型的标识符
   - **使用场景**（必填）：选择使用场景
     - **智能问答**：仅用于智能问答
     - **知识库问答**：仅用于知识库问答
     - **两者都支持**：两种场景都支持
   - **是否启用**：是否启用该模型
3. 点击"确定"添加模型

**编辑模型**：
1. 在模型列表中，找到要编辑的模型
2. 点击"编辑"按钮
3. 修改模型信息
4. 点击"确定"保存修改

**删除模型**：
1. 在模型列表中，找到要删除的模型
2. 点击"删除"按钮
3. 确认删除操作

> **注意**：不能删除正在使用的模型，删除前请确保没有应用或知识库使用该模型。

**设置默认模型**：
1. 在模型列表中，找到要设置为默认的模型
2. 点击"设为默认"按钮
3. 选择使用场景（智能问答/知识库问答）
4. 确认操作

**启用/禁用模型**：
1. 在模型列表中，找到要操作的模型
2. 切换"启用"开关
3. 系统会自动保存修改

**测试模型连接**：
1. 在模型列表中，找到要测试的模型
2. 点击"测试连接"按钮
3. 系统会测试模型连接是否正常

### 3.9 Text2SQL

Text2SQL 功能允许您使用自然语言查询数据库，系统会自动将自然语言转换为 SQL 或 MongoDB 查询语句并执行。

#### 3.9.1 开始使用

1. 在左侧菜单中点击"SQL 生成"
2. 选择要查询的数据源
3. 选择要查询的表（可选，不选择则查询所有表）
4. 在输入框中输入您的查询需求（自然语言）
5. 选择问答模型（可选，不选择则使用默认模型）
6. 点击"生成"按钮

#### 3.9.2 查询功能

**支持的数据库类型**：
- **PostgreSQL**：关系型数据库
- **MySQL**：关系型数据库
- **Oracle**：关系型数据库
- **MongoDB**：文档型数据库

**查询流程**：
1. 系统自动获取数据库表结构（支持缓存）
2. 使用 LLM 将自然语言转换为 SQL/MongoDB 查询
3. 执行查询并返回结果
4. 显示生成的 SQL/MongoDB 查询语句和查询结果

#### 3.9.3 表结构管理

**查看表列表**：
1. 选择数据源后，系统会自动加载表列表
2. 表列表显示所有可用的表
3. 可以选择一个或多个表进行查询

**查看表结构**：
1. 在表列表中，点击表名
2. 系统会显示该表的详细结构，包括：
   - 字段名称
   - 字段类型
   - 是否主键
   - 是否可空
   - 默认值
   - 注释

**刷新表结构**：
1. 如果数据库结构发生变化，可以点击"刷新表结构"按钮
2. 系统会重新获取最新的表结构信息
3. 可以选择强制刷新（清除缓存）

#### 3.9.4 查询结果

**结果展示**：
- 生成的 SQL/MongoDB 查询语句会显示在代码框中
- 查询结果以表格形式展示
- 支持结果导出（未来功能）

**统计查询**：
- 支持聚合查询（COUNT、SUM、AVG 等）
- 支持分组查询（GROUP BY）
- 支持排序和限制（ORDER BY、LIMIT）

#### 3.9.5 注意事项

- 确保数据源连接正常
- 确保有足够的数据库权限
- 复杂查询可能需要较长时间
- 建议先查看表结构，了解数据库结构后再查询

1. 在系统配置页面，点击"添加配置"按钮
2. 填写配置信息：
   - **配置键**（必填）：配置的唯一标识，例如：`drawio.defaultModelId`
   - **配置值**（必填）：配置的值
   - **配置分组**（可选）：配置的分组，例如：`help`、`system`
   - **配置类型**（可选）：配置的类型，例如：`string`、`number`、`boolean`
   - **描述**（可选）：配置的说明
3. 点击"确定"添加配置

#### 3.12.3 编辑配置

1. 在配置列表中，找到要编辑的配置
2. 点击"编辑"按钮
3. 修改配置值或其他信息（注意：配置键不可修改）
4. 点击"确定"保存修改

#### 3.12.4 删除配置

1. 在配置列表中，找到要删除的配置
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除配置可能会影响系统功能，请谨慎操作！

#### 3.12.5 常用配置说明

**AI 绘图默认模型**：
- 配置键：`drawio.defaultModelId`
- 配置值：模型 ID（数字）
- 说明：设置 AI 绘图功能的默认模型

**帮助配置**：
- 配置键：`help.knowledgeBaseId`
- 配置值：知识库 ID（数字）
- 说明：设置帮助功能使用的知识库

**全局主题色配置**：
- 配置键：`global.theme`
- 配置值：主题名称（如：`blue`、`green`、`purple` 等）
- 说明：设置系统全局主题色

## 系统配置

### 4.1 系统参数配置

系统的主要配置在 `application.yml` 文件中，包括：

- **数据库配置**：PostgreSQL 连接信息
- **MinIO 配置**：对象存储配置
- **JWT 配置**：Token 密钥和过期时间
- **Dify 配置**：Dify API 基础配置
- **RAG 配置**：文档分块、检索参数等

> **注意**：修改配置文件后需要重启应用才能生效。

### 4.2 模型配置

大模型和向量数据库的配置已迁移到数据库，可以通过管理端界面进行配置，无需修改配置文件。

### 4.3 权限配置

系统支持基于角色的访问控制（RBAC）：

- **管理员**：拥有所有权限
- **普通用户**：只能访问被授权的资源

权限管理通过用户管理界面进行配置。

## 常见问题

### Q1: 如何创建第一个应用？

**A**: 
1. 确保您已经配置好 Dify 平台
2. 在 Dify 平台中创建应用，获取应用 ID 和 API Key
3. 在管理端"应用管理"页面，点击"创建应用"
4. 填写应用信息，包括 Dify 应用 ID 和 API Key
5. 保存后即可使用

### Q2: 新用户注册后无法登录？

**A**: 新用户注册后状态为"待审核"，需要管理员审核后才能登录。请在"用户管理"页面审核该用户。

### Q3: 如何为用户分配应用权限？

**A**: 
1. 在"用户管理"页面，找到要管理的用户
2. 点击"应用管理"按钮
3. 在应用权限管理界面中，切换应用的可见性

### Q4: 文档向量化失败怎么办？

**A**: 
1. 检查文档格式是否支持
2. 检查向量化模型是否正常
3. 检查向量数据库连接是否正常
4. 在文档管理页面，点击"重新向量化"按钮重试

### Q5: 如何配置问答模型？

**A**: 
1. 在"大模型管理"页面，切换到"问答模型"标签页
2. 点击"添加模型"按钮
3. 填写模型信息（提供商、API 地址、API Key、模型标识等）
4. 选择使用场景（智能问答/知识库问答/两者都支持）
5. 保存后即可使用

### Q6: 如何选择向量存储类型？

**A**: 
- **开发测试环境**：使用 FAISS（无需额外服务）
- **生产环境**：使用 Qdrant 或 Milvus（分布式、高性能）

选择建议：
- 如果资源有限，使用 FAISS
- 如果需要高性能和可扩展性，使用 Qdrant 或 Milvus

### Q7: 知识库创建后可以修改向量存储类型吗？

**A**: 不可以。知识库创建后，向量存储类型无法修改。如果需要使用不同的向量存储类型，需要创建新的知识库。

### Q8: 如何备份向量数据？

**A**: 
- **Qdrant**：备份 Qdrant 数据目录
- **Milvus**：备份 Milvus 数据目录
- **FAISS**：备份 `data/faiss` 目录

建议定期备份向量数据，以防止数据丢失。

### Q9: 如何监控系统使用情况？

**A**: 
1. 在"会话历史"页面查看对话统计
2. 查看应用使用情况
3. 查看知识库文档数量
4. 查看用户活跃情况

### Q10: 如何重置管理员密码？

**A**: 
1. 在"用户管理"页面，找到管理员用户
2. 点击"重置密码"按钮
3. 输入新密码并确认
4. 使用新密码登录

### Q11: 如何配置多个向量数据库实例？

**A**: 
1. 在"向量数据库管理"页面，添加多个配置
2. 每个配置可以指向不同的实例
3. 创建知识库时选择要使用的配置
4. 可以设置一个默认配置

### Q12: 问答模型和向量化模型有什么区别？

**A**: 
- **问答模型**：用于生成答案，支持智能问答和知识库问答
- **向量化模型**：用于将文档转换为向量，用于知识库检索

两者是不同的模型，需要分别配置。

### Q13: 如何测试模型连接？

**A**: 
1. 在"大模型管理"页面，找到要测试的模型
2. 点击"测试连接"按钮
3. 系统会测试模型连接是否正常
4. 如果连接失败，检查 API 地址、API Key 等配置是否正确

### Q14: 如何设置默认模型？

**A**: 
1. 在"大模型管理"页面，找到要设置为默认的模型
2. 点击"设为默认"按钮
3. 选择使用场景（问答模型需要选择场景）
4. 确认操作

### Q15: 如何禁用模型？

**A**: 
1. 在"大模型管理"页面，找到要禁用的模型
2. 切换"启用"开关为关闭状态
3. 系统会自动保存修改

禁用后，该模型将不再可用，但不会影响已创建的配置。

### Q16: 如何使用 Text2SQL 功能？

**A**: 
1. 首先在"数据源管理"页面创建数据源
2. 在"SQL 生成"页面选择数据源
3. 输入自然语言查询需求
4. 系统会自动生成 SQL 并执行查询

### Q17: Text2SQL 支持哪些数据库？

**A**: 系统支持以下数据库：
- PostgreSQL
- MySQL
- Oracle
- MongoDB

### Q18: 如何配置 AI 绘图默认模型？

**A**: 
1. 在"系统配置"页面，添加或编辑配置
2. 配置键：`drawio.defaultModelId`
3. 配置值：模型 ID（数字）
4. 保存后，AI 绘图功能将使用该模型作为默认模型

### Q19: 如何管理数据源？

**A**: 
1. 在"数据源管理"页面，可以创建、编辑、删除数据源
2. 创建数据源时，建议先测试连接
3. 确保数据库用户有足够的权限
4. 表结构会自动缓存，提高查询性能

### Q20: 系统配置的作用是什么？

**A**: 系统配置用于管理系统级别的参数，如：
- AI 绘图默认模型
- 帮助功能配置
- 全局主题色
- 其他系统参数

这些配置可以通过界面管理，无需修改配置文件。

### Q21: 向量数据库支持哪些类型？

**A**: 系统支持以下向量数据库：
- **Qdrant**：分布式向量数据库，适合生产环境
- **Milvus**：分布式向量数据库，适合生产环境
- **FAISS**：本地文件存储，适合开发测试环境
- **Chroma**：轻量级向量数据库，适合中小规模应用
- **Weaviate**：开源向量数据库，支持多种向量化模型
- **Elasticsearch**：分布式搜索和分析引擎，支持向量搜索

### Q22: 如何选择向量数据库？

**A**: 选择建议：
- **开发测试环境**：使用 FAISS（无需额外服务）
- **中小规模应用**：使用 Chroma
- **生产环境**：使用 Qdrant、Milvus 或 Weaviate
- **需要搜索功能**：使用 Elasticsearch
