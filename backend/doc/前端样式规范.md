### 4.4 加载状态管理最佳实践

#### 2.4.1 基础加载模式

```javascript
// 定义加载状态
const loading = ref(false)

// 加载数据
async function fetchData() {
  loading.value = true
  try {
    const response = await apiCall()
    // 处理数据
  } catch (error) {
    ElMessage.error('加载失败')
  } finally {
    loading.value = false
  }
}
```

#### 2.4.2 多加载状态管理

```javascript
// 多个独立的加载状态
const loading = reactive({
  data: false,
  submit: false,
  delete: false
})

// 使用示例
async function loadData() {
  loading.data = true
  try {
    // ...
  } finally {
    loading.data = false
  }
}
```

#### 2.4.3 列表项加载状态

```javascript
// 为每个列表项管理加载状态
const reindexingDocIds = ref([])

// 检查是否正在加载
const isReindexing = (docId) => reindexingDocIds.value.includes(docId)

// 开始加载
reindexingDocIds.value.push(docId)

// 结束加载
const index = reindexingDocIds.value.indexOf(docId)
if (index > -1) {
  reindexingDocIds.value.splice(index, 1)
}

// 模板中使用
<el-button 
  :loading="isReindexing(row.id)"
  @click="handleReindex(row)"
>
  重新索引
</el-button>
```

#### 2.4.4 懒加载分段内容

```javascript
// 已加载的分段
const loadedSegments = ref(new Set())

// 正在加载的分段
const loadingSegments = ref(new Set())

// 分段内容
const segmentTranslations = ref([])

// 加载单个分段
async function loadSegment(index) {
  // 防止重复加载
  if (loadingSegments.value.has(index)) {
    return
  }
  
  // 检查是否已加载
  if (loadedSegments.value.has(index)) {
    return
  }
  
  loadingSegments.value.add(index)
  
  try {
    const content = await fetchSegmentContent(index)
    segmentTranslations.value[index] = content
    loadedSegments.value.add(index)
  } finally {
    loadingSegments.value.delete(index)
  }
}
```

### 4.5 加载动画性能优化

#### 2.5.1 使用硬件加速

```css
/* 激活 GPU 加速 */
.optimized-animation {
  transform: translateZ(0);
  will-change: transform, opacity;
}

/* 或使用 */
.optimized-animation {
  backface-visibility: hidden;
  perspective: 1000px;
}
```

#### 2.5.2 避免引起重排的属性

```css
/* 推荐 - 使用 transform 和 opacity */
.good-animation {
  transform: translateX(100px);
  opacity: 0.5;
}

/* 避免 - 引起重排 */
.bad-animation {
  left: 100px;
  top: 50px;
  width: 200px;
}
```

#### 2.5.3 使用 requestAnimationFrame

```javascript
// 使用 requestAnimationFrame 优化动画更新
let rafId = null
let pendingContent = null

const scheduleUpdate = (content) => {
  pendingContent = content
  
  // 取消之前的更新
  if (rafId !== null) {
    cancelAnimationFrame(rafId)
  }
  
  // 在下一次重绘前更新
  rafId = requestAnimationFrame(() => {
    if (pendingContent !== null) {
      // 更新 DOM
      updateContent(pendingContent)
    }
    rafId = null
  })
}
```

### 4.6 加载状态 UX 建议

1. **即时反馈**: 用户操作后立即显示加载状态
2. **明确提示**: 使用清晰的文字说明加载内容
3. **合理时长**: 避免长时间加载，考虑分页或懒加载
4. **视觉层次**: 加载状态不应遮挡重要内容
5. **错误处理**: 加载失败时提供明确的错误信息和重试机制

4. **懒加载**

```vue
<el-image lazy :src="imageUrl" />
```

5. **CSS 变量性能**
```css
/* ✅ 推荐：使用 CSS 变量，浏览器会缓存计算值 */
.element {
  color: var(--color-primary);
  padding: var(--spacing-md);
}

/* ❌ 避免：频繁计算的值 */
.element {
  color: rgb(64, 158, 255); /* 每次都需要解析 */
  padding: calc(16px * 1.5); /* 每次都需要计算 */
}
```

6. **动画性能优化**
```css
/* ✅ 使用 transform 和 opacity（GPU 加速） */
.animated {
  transform: translateX(100px);
  opacity: 0.5;
  will-change: transform, opacity;
}

/* ❌ 避免使用会引起重排的属性 */
.animated {
  left: 100px; /* 引起重排 */
  width: 200px; /* 引起重排 */
}
```

## 二十、注意事项

1. **设计令牌优先**: 始终使用设计令牌（CSS 变量）而非硬编码值
   - ✅ `var(--color-primary)` 
   - ❌ `#409EFF`
   - ✅ `var(--spacing-md)`
   - ❌ `16px`

2. **响应式设计**: 所有页面都需要考虑移动端适配，使用媒体查询和响应式单位

3. **加载状态**: 所有异步操作都需要显示加载状态，使用统一的加载组件

4. **错误处理**: 所有 API 调用都需要错误处理，使用统一的错误提示

5. **表单验证**: 所有表单都需要完整的验证规则，使用 Element Plus 的表单验证

6. **无障碍访问**: 
   - 为图片添加 `alt` 属性
   - 为按钮添加 `title` 或 `aria-label` 属性
   - 使用语义化 HTML 标签

7. **浏览器兼容**: 
   - CSS 变量在现代浏览器中支持良好
   - 如需支持旧浏览器，提供 fallback 值

8. **动画性能**: 
   - 优先使用 `transform` 和 `opacity` 进行动画（GPU 加速）
   - 避免使用会引起重排的属性（`width`, `height`, `left`, `top` 等）
   - 使用 `will-change` 提示浏览器优化

9. **样式作用域**: 
   - 组件样式使用 `scoped` 避免污染全局样式
   - 深度选择器使用 `:deep()` 而非 `/deep/` 或 `>>>`

10. **代码组织**: 
    - 按功能模块组织样式
    - 使用 BEM 命名规范或语义化类名
    - 避免过深的嵌套（建议不超过 3 层）
