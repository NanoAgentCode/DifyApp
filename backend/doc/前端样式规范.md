### 2.4 加载状态管理最佳实践

#### 2.4.1 基础加载模式

```javascript
// 定义加载状态
const loading = ref(false)

// 加载数据
async function fetchData() {
  loading.value = true
  try {
    const response = await apiCall()
    // 处理数据
  } catch (error) {
    ElMessage.error('加载失败')
  } finally {
    loading.value = false
  }
}
```

#### 2.4.2 多加载状态管理

```javascript
// 多个独立的加载状态
const loading = reactive({
  data: false,
  submit: false,
  delete: false
})

// 使用示例
async function loadData() {
  loading.data = true
  try {
    // ...
  } finally {
    loading.data = false
  }
}
```

#### 2.4.3 列表项加载状态

```javascript
// 为每个列表项管理加载状态
const reindexingDocIds = ref([])

// 检查是否正在加载
const isReindexing = (docId) => reindexingDocIds.value.includes(docId)

// 开始加载
reindexingDocIds.value.push(docId)

// 结束加载
const index = reindexingDocIds.value.indexOf(docId)
if (index > -1) {
  reindexingDocIds.value.splice(index, 1)
}

// 模板中使用
<el-button 
  :loading="isReindexing(row.id)"
  @click="handleReindex(row)"
>
  重新索引
</el-button>
```

#### 2.4.4 懒加载分段内容

```javascript
// 已加载的分段
const loadedSegments = ref(new Set())

// 正在加载的分段
const loadingSegments = ref(new Set())

// 分段内容
const segmentTranslations = ref([])

// 加载单个分段
async function loadSegment(index) {
  // 防止重复加载
  if (loadingSegments.value.has(index)) {
    return
  }
  
  // 检查是否已加载
  if (loadedSegments.value.has(index)) {
    return
  }
  
  loadingSegments.value.add(index)
  
  try {
    const content = await fetchSegmentContent(index)
    segmentTranslations.value[index] = content
    loadedSegments.value.add(index)
  } finally {
    loadingSegments.value.delete(index)
  }
}
```

### 2.5 加载动画性能优化

#### 2.5.1 使用硬件加速

```css
/* 激活 GPU 加速 */
.optimized-animation {
  transform: translateZ(0);
  will-change: transform, opacity;
}

/* 或使用 */
.optimized-animation {
  backface-visibility: hidden;
  perspective: 1000px;
}
```

#### 2.5.2 避免引起重排的属性

```css
/* 推荐 - 使用 transform 和 opacity */
.good-animation {
  transform: translateX(100px);
  opacity: 0.5;
}

/* 避免 - 引起重排 */
.bad-animation {
  left: 100px;
  top: 50px;
  width: 200px;
}
```

#### 2.5.3 使用 requestAnimationFrame

```javascript
// 使用 requestAnimationFrame 优化动画更新
let rafId = null
let pendingContent = null

const scheduleUpdate = (content) => {
  pendingContent = content
  
  // 取消之前的更新
  if (rafId !== null) {
    cancelAnimationFrame(rafId)
  }
  
  // 在下一次重绘前更新
  rafId = requestAnimationFrame(() => {
    if (pendingContent !== null) {
      // 更新 DOM
      updateContent(pendingContent)
    }
    rafId = null
  })
}
```

### 2.6 加载状态 UX 建议

1. **即时反馈**: 用户操作后立即显示加载状态
2. **明确提示**: 使用清晰的文字说明加载内容
3. **合理时长**: 避免长时间加载，考虑分页或懒加载
4. **视觉层次**: 加载状态不应遮挡重要内容
5. **错误处理**: 加载失败时提供明确的错误信息和重试机制

## 十八、注意事项

1. **主题色使用**: 始终使用 CSS 变量 `var(--el-color-primary)` 而非硬编码颜色值
2. **响应式**: 所有页面都需要考虑移动端适配
3. **加载状态**: 所有异步操作都需要显示加载状态
4. **错误处理**: 所有 API 调用都需要错误处理
5. **表单验证**: 所有表单都需要完整的验证规则
6. **无障碍**: 为图片添加 alt 属性，为按钮添加 title 属性
7. **浏览器兼容**: 使用 CSS 变量时提供 fallback
8. **动画性能**: 使用 transform 和 opacity 进行动画，避免引起重排
