# DifyApp 外部依赖部署手册

## 1. 概述

本文档介绍 DifyApp 系统所需的外部依赖服务的部署方法。系统依赖包括：

- **关系型数据库**：PostgreSQL（必需）
- **对象存储**：RustFS（必需，S3兼容，MinIO替代品）
- **向量数据库**：Qdrant、Milvus、FAISS、Chroma、Weaviate、PgVector、Elasticsearch（至少选择一种）
- **OCR服务**：EasyOCR（可选，但推荐）
- **思维导图服务**：Mindmap（可选）
- **缓存服务**：Redis（可选，但推荐）

> **注意**：FAISS 是本地文件存储，无需 Docker 部署。PgVector 是 PostgreSQL 扩展，无需单独部署。

## 2. 快速部署

### 2.1 使用 Docker Compose 一键部署

系统提供了完整的 `docker-compose.yml` 文件，包含所有外部依赖服务。您可以根据实际需求选择部署哪些服务。

**部署步骤**：

1. 克隆或下载项目代码
2. 进入项目根目录
3. 根据需要修改 `docker-compose.dependencies.yml` 文件
4. 启动服务：

```bash
# 启动所有依赖服务
docker-compose -f docker-compose.dependencies.yml up -d

# 查看服务状态
docker-compose -f docker-compose.dependencies.yml ps

# 查看服务日志
docker-compose -f docker-compose.dependencies.yml logs -f
```

### 2.2 服务端口说明

| 服务 | 端口 | 说明 |
|------|------|------|
| PostgreSQL | 15432 | 数据库服务 |
| RustFS API | 9000 | 对象存储API（S3兼容） |
| RustFS Console | 9001 | 对象存储控制台 |
| Redis | 16379 | 缓存服务 |
| Qdrant HTTP | 6333 | Qdrant HTTP API |
| Qdrant gRPC | 6334 | Qdrant gRPC API |
| Milvus | 19530 | Milvus gRPC API |
| Chroma | 8002 | Chroma API |
| Weaviate | 8080 | Weaviate API |
| Elasticsearch | 9200 | Elasticsearch HTTP API |
| EasyOCR | 8001 | OCR服务API |
| Mindmap | 6066 | 思维导图服务API |

> **注意**：端口冲突处理：如果端口已被占用，请修改 `docker-compose.dependencies.yml` 中的端口映射。

## 3. 必需服务部署

### 3.1 PostgreSQL 数据库

PostgreSQL 是系统的主数据库，存储所有业务数据。

#### 3.1.1 Docker 部署

```bash
docker run -d \
  --name postgres \
  -p 15432:5432 \
  -e POSTGRES_PASSWORD=123456 \
  -e POSTGRES_DB=postgres \
  -e POSTGRES_USER=postgres \
  -v postgres-data:/var/lib/postgresql/data \
  --restart unless-stopped \
  ankane/pgvector:latest
```

#### 3.1.2 启用 PgVector 扩展

如果使用 PgVector 作为向量数据库，需要启用扩展：

```sql
-- 连接到数据库
psql -h localhost -p 15432 -U postgres -d postgres

-- 启用 pgvector 扩展
CREATE EXTENSION IF NOT EXISTS vector;
```

#### 3.1.3 配置说明

- **数据库名**：`postgres`（默认）
- **用户名**：`postgres`（默认）
- **密码**：`123456`（请修改为强密码）
- **端口**：`15432`（可自定义）

#### 3.1.4 数据持久化

数据存储在 Docker 卷 `postgres-data` 中，即使容器删除数据也不会丢失。

**备份数据**：
```bash
docker exec postgres pg_dump -U postgres postgres > backup.sql
```

**恢复数据**：
```bash
docker exec -i postgres psql -U postgres postgres < backup.sql
```

### 3.2 RustFS 对象存储

RustFS 用于存储文档文件。RustFS 是一个高性能的分布式对象存储系统，100% 兼容 S3 协议，可以作为 MinIO 的替代品。

#### 3.2.1 Docker 部署

```bash
docker run -d \
  --name rustfs \
  -p 9000:9000 \
  -p 9001:9001 \
  -e RUSTFS_ACCESS_KEY=rustfsadmin \
  -e RUSTFS_SECRET_KEY=rustfsadmin \
  -e RUSTFS_CONSOLE_ENABLE=true \
  -e RUSTFS_ADDRESS=:9000 \
  -v rustfs-data:/data \
  --restart unless-stopped \
  rustfs/rustfs:latest \
  /data
```

#### 3.2.2 访问控制台

1. 打开浏览器访问：`http://localhost:9001`
2. 使用默认账号登录：
   - 用户名：`rustfsadmin`
   - 密码：`rustfsadmin`
3. 创建存储桶（Bucket）：
   - 点击 "Create Bucket"
   - 输入存储桶名称（如：`knowledge-base`）
   - 点击 "Create Bucket"

#### 3.2.3 配置说明

- **API地址**：`http://localhost:9000`（S3兼容）
- **控制台地址**：`http://localhost:9001`
- **默认访问密钥**：`rustfsadmin`
- **默认密钥**：`rustfsadmin`（请修改为强密码）
- **数据目录权限**：需要设置 uid:gid = 10001:10001（如果使用本地目录挂载）

#### 3.2.4 数据持久化

数据存储在 Docker 卷 `rustfs-data` 中。

**如果使用本地目录挂载**：
```bash
# 创建数据目录
mkdir -p ./rustfs/data

# 设置目录权限（RustFS容器以uid 10001运行）
chown -R 10001:10001 ./rustfs/data
```

#### 3.2.5 RustFS 特性

- ✅ 100% S3 协议兼容
- ✅ 高性能（基于 Rust 开发）
- ✅ 内存安全
- ✅ 商业友好（Apache 2.0 许可证）
- ✅ 跨平台支持
- ✅ 云原生部署

## 4. 向量数据库部署

系统支持多种向量数据库，您可以根据实际需求选择一种或多种部署。

### 4.1 Qdrant（推荐用于生产环境）

Qdrant 是高性能分布式向量数据库，适合生产环境。

#### 4.1.1 Docker 部署

```bash
docker run -d \
  --name qdrant \
  -p 6333:6333 \
  -p 6334:6334 \
  -v qdrant-data:/qdrant/storage \
  --restart unless-stopped \
  qdrant/qdrant
```

#### 4.1.2 访问控制台

打开浏览器访问：`http://localhost:6333/dashboard`

#### 4.1.3 配置说明

- **HTTP API**：`http://localhost:6333`
- **gRPC API**：`localhost:6334`
- **数据持久化**：存储在 Docker 卷 `qdrant-data` 中

#### 4.1.4 集群部署（可选）

对于生产环境，可以部署 Qdrant 集群。参考 [Qdrant 官方文档](https://qdrant.tech/documentation/guides/clustering/)。

### 4.2 Milvus

Milvus 是开源向量数据库，支持大规模向量检索。

#### 4.2.1 Docker Compose 部署（推荐）

Milvus 需要多个组件（etcd、MinIO），建议使用 Docker Compose 部署：

```yaml
version: '3.8'

services:
  etcd:
    image: quay.io/coreos/etcd:v3.5.5
    container_name: milvus-etcd
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
      - ETCD_SNAPSHOT_COUNT=50000
    volumes:
      - etcd-data:/etcd
    command: etcd -advertise-client-urls=http://127.0.0.1:2379 -listen-client-urls http://0.0.0.0:2379 --data-dir /etcd
    healthcheck:
      test: ["CMD", "etcdctl", "endpoint", "health"]
      interval: 30s
      timeout: 20s
      retries: 3

  minio:
    image: minio/minio:RELEASE.2023-03-20T20-16-18Z
    container_name: milvus-minio
    environment:
      MINIO_ACCESS_KEY: minioadmin
      MINIO_SECRET_KEY: minioadmin
    ports:
      - "9002:9000"
      - "9003:9001"
    volumes:
      - minio-data:/minio_data
    command: minio server /minio_data --console-address ":9001"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 20s
      retries: 3

  milvus:
    image: milvusdb/milvus:v2.3.3
    container_name: milvus-standalone
    command: ["milvus", "run", "standalone"]
    environment:
      ETCD_ENDPOINTS: etcd:2379
      MINIO_ADDRESS: minio:9000
    volumes:
      - milvus-data:/var/lib/milvus
    ports:
      - "19530:19530"
      - "9091:9091"
    depends_on:
      - "etcd"
      - "minio"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9091/healthz"]
      interval: 30s
      start_period: 90s
      timeout: 20s
      retries: 3

volumes:
  etcd-data:
  milvus-storage-data:
  milvus-data:
```

#### 4.2.2 配置说明

- **gRPC API**：`localhost:19530`
- **HTTP API**：`http://localhost:9091`
- **数据持久化**：存储在 Docker 卷中

### 4.3 FAISS

FAISS 是本地文件存储，无需 Docker 部署。系统会自动在本地文件系统中创建 FAISS 索引文件。

**配置说明**：
- **存储路径**：默认 `./data/faiss`（相对于应用运行目录）
- **数据格式**：每个知识库的向量数据存储在独立目录中：`{basePath}/kb_{knowledgeBaseId}/`
- **元数据文件**：`metadata.json`

**注意事项**：
- FAISS 数据存储在本地文件系统，需要确保应用有读写权限
- 建议定期备份 `data/faiss` 目录
- 不支持分布式部署

### 4.4 Chroma

Chroma 是轻量级向量数据库，适合中小规模应用。

#### 4.4.1 Docker 部署

```bash
docker run -d \
  --name chroma \
  -p 8000:8000 \
  -v chroma-data:/chroma/chroma \
  --restart unless-stopped \
  ghcr.io/chroma-core/chroma:latest
```

#### 4.4.2 配置说明

- **API地址**：`http://localhost:8002`
- **数据持久化**：存储在 Docker 卷 `chroma-data` 中

### 4.5 Weaviate

Weaviate 是开源向量数据库，支持多种向量化模型。

#### 4.5.1 Docker Compose 部署

```yaml
version: '3.8'

services:
  weaviate:
    image: semitechnologies/weaviate:latest
    container_name: weaviate
    ports:
      - "8080:8080"
    environment:
      - QUERY_DEFAULTS_LIMIT=25
      - AUTHENTICATION_ANONYMOUS_ACCESS_ENABLED=true
      - PERSISTENCE_DATA_PATH=/var/lib/weaviate
      - DEFAULT_VECTORIZER_MODULE=none
      - ENABLE_MODULES=text2vec-openai,text2vec-cohere,text2vec-huggingface
      - CLUSTER_HOSTNAME=node1
    volumes:
      - weaviate-data:/var/lib/weaviate
    restart: unless-stopped

volumes:
  weaviate-data:
```

#### 4.5.2 配置说明

- **API地址**：`http://localhost:8080`
- **数据持久化**：存储在 Docker 卷 `weaviate-data` 中

### 4.6 PgVector

PgVector 是 PostgreSQL 的扩展，无需单独部署。只需在 PostgreSQL 中启用扩展即可。

**启用步骤**：
1. 连接到 PostgreSQL 数据库
2. 执行以下 SQL：

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

**配置说明**：
- 使用与主数据库相同的 PostgreSQL 实例
- 无需额外端口
- 数据存储在 PostgreSQL 数据库中

### 4.7 Elasticsearch

Elasticsearch 是分布式搜索和分析引擎，支持向量搜索。

#### 4.7.1 Docker Compose 部署

```yaml
version: '3.8'

services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.11.0
    container_name: elasticsearch
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
    ports:
      - "9200:9200"
      - "9300:9300"
    volumes:
      - elasticsearch-data:/usr/share/elasticsearch/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:9200/_cluster/health || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  elasticsearch-data:
```

#### 4.7.2 配置说明

- **HTTP API**：`http://localhost:9200`
- **数据持久化**：存储在 Docker 卷 `elasticsearch-data` 中
- **内存要求**：建议至少 512MB 内存

## 5. OCR 服务部署

### 5.1 EasyOCR 服务

EasyOCR 用于图片和PDF文件的文字识别。

#### 5.1.1 Docker Compose 部署

```yaml
version: '3.8'

services:
  easyocr:
    build:
      context: ./easy_ocr
      dockerfile: Dockerfile
    image: difyapp/easyocr:latest
    container_name: easyocr-service
    ports:
      - "8000:8000"
    environment:
      - PORT=8000
      - HOST=0.0.0.0
    volumes:
      # 挂载模型目录以加速后续启动（模型会下载到容器内）
      - easyocr-models:/root/.EasyOCR
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 120s  # 首次启动需要下载模型，给足够时间
    # 资源限制（根据实际情况调整）
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 4G
        reservations:
          cpus: '1'
          memory: 2G

volumes:
  easyocr-models:
    driver: local
```

#### 5.1.2 配置说明

- **API地址**：`http://localhost:8001`
- **健康检查**：`http://localhost:8001/health`
- **模型存储**：存储在 Docker 卷 `easyocr-models` 中
- **资源要求**：建议至少 2GB 内存

#### 5.1.3 首次启动

首次启动时，EasyOCR 会自动下载模型文件，可能需要较长时间（10-30分钟，取决于网络速度）。模型下载完成后，后续启动会更快。

#### 5.1.4 在系统配置中设置OCR服务地址

1. 登录管理端
2. 进入"系统配置"页面
3. 添加或编辑配置：
   - **配置键**：`ocr.service.url`
   - **配置值**：`http://localhost:8001`
   - **配置分组**：`ocr`
4. 可选配置：`ocr.service.timeout`（超时时间，毫秒，默认：30000）

## 6. 思维导图服务部署

```yaml
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:6066/"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 30s
```

#### 6.1.2 配置说明

- **API地址**：`http://localhost:6066`
- **数据持久化**：静态文件存储在 `./mindmap/static` 目录中

## 7. 缓存服务部署

### 7.1 Redis

Redis 用于缓存用户信息、模型配置等，提升系统性能。

#### 7.1.1 Docker 部署

```bash
docker run -d \
  --name redis \
  -p 16379:6379 \
  -v redis-data:/data \
  --restart unless-stopped \
  redis:7-alpine \
  redis-server --appendonly yes
```

#### 7.1.2 配置说明

- **连接地址**：`localhost:16379`
- **数据持久化**：AOF 模式，数据存储在 Docker 卷 `redis-data` 中
- **密码**：默认无密码（生产环境建议设置密码）

#### 7.1.3 设置密码（可选）

```bash
docker run -d \
  --name redis \
  -p 16379:6379 \
  -v redis-data:/data \
  --restart unless-stopped \
  redis:7-alpine \
  redis-server --appendonly yes --requirepass yourpassword
```

## 8. 完整 Docker Compose 配置

系统提供了完整的 `docker-compose.dependencies.yml` 文件，包含所有外部依赖服务。您可以根据实际需求启用或禁用某些服务。

### 8.1 完整配置文件

创建 `docker-compose.dependencies.yml` 文件：

```yaml
version: '3.8'

services:
  # ==================== 必需服务 ====================
  
  # PostgreSQL 数据库
  postgres:
    image: ankane/pgvector:latest
    container_name: difyapp-postgres
    ports:
      - "15432:5432"
    environment:
      - POSTGRES_PASSWORD=123456
      - POSTGRES_DB=postgres
      - POSTGRES_USER=postgres
    volumes:
      - postgres-data:/var/lib/postgresql/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  # RustFS 对象存储（S3兼容，MinIO替代品）
  rustfs:
    image: rustfs/rustfs:latest
    container_name: difyapp-rustfs
    ports:
      - "9000:9000"  # S3 API端口
      - "9001:9001"  # 控制台端口
    environment:
      - RUSTFS_ACCESS_KEY=rustfsadmin
      - RUSTFS_SECRET_KEY=rustfsadmin
      - RUSTFS_CONSOLE_ENABLE=true
      - RUSTFS_ADDRESS=:9000
    volumes:
      - rustfs-data:/data
    command:
      - /data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 20s
      retries: 3
      start_period: 10s
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '0.5'
          memory: 512M

  # ==================== 可选服务 ====================
  
  # Redis 缓存（推荐）
  redis:
    image: redis:7-alpine
    container_name: difyapp-redis
    ports:
      - "16379:6379"
    volumes:
      - redis-data:/data
    command: redis-server --appendonly yes
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Qdrant 向量数据库
  qdrant:
    image: qdrant/qdrant:latest
    container_name: difyapp-qdrant
    ports:
      - "6333:6333"
      - "6334:6334"
    volumes:
      - qdrant-data:/qdrant/storage
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:6333/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Milvus 向量数据库（需要 etcd 和 minio）
  milvus-etcd:
    image: quay.io/coreos/etcd:v3.5.5
    container_name: difyapp-milvus-etcd
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
      - ETCD_SNAPSHOT_COUNT=50000
    volumes:
      - milvus-etcd-data:/etcd
    command: etcd -advertise-client-urls=http://127.0.0.1:2379 -listen-client-urls http://0.0.0.0:2379 --data-dir /etcd
    healthcheck:
      test: ["CMD", "etcdctl", "endpoint", "health"]
      interval: 30s
      timeout: 20s
      retries: 3

  milvus-minio:
    image: minio/minio:RELEASE.2023-03-20T20-16-18Z
    container_name: difyapp-milvus-minio
    environment:
      MINIO_ACCESS_KEY: minioadmin
      MINIO_SECRET_KEY: minioadmin
    ports:
      - "9002:9000"
      - "9003:9001"
    volumes:
      - milvus-minio-data:/minio_data
    command: minio server /minio_data --console-address ":9001"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 20s
      retries: 3

  milvus:
    image: milvusdb/milvus:v2.3.3
    container_name: difyapp-milvus
    command: ["milvus", "run", "standalone"]
    environment:
      ETCD_ENDPOINTS: milvus-etcd:2379
      MINIO_ADDRESS: milvus-minio:9000
    volumes:
      - milvus-data:/var/lib/milvus
    ports:
      - "19530:19530"
      - "9091:9091"
    depends_on:
      milvus-etcd:
        condition: service_healthy
      milvus-minio:
        condition: service_healthy
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9091/healthz"]
      interval: 30s
      start_period: 90s
      timeout: 20s
      retries: 3

  # Chroma 向量数据库
  chroma:
    image: ghcr.io/chroma-core/chroma:latest
    container_name: difyapp-chroma
    ports:
      - "8002:8000"  # 使用 8002 端口避免与其他服务冲突
    volumes:
      - chroma-data:/chroma/chroma
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/api/v1/heartbeat"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Weaviate 向量数据库
  weaviate:
    image: semitechnologies/weaviate:latest
```

## 11. 常见问题

### Q1: 端口冲突怎么办？

**A**: 如果端口已被占用，可以修改 `docker-compose.dependencies.yml` 中的端口映射。例如：

```yaml
ports:
  - "15433:5432"  # 将外部端口改为 15433
```

### Q2: 如何备份数据？

**A**: 

**PostgreSQL 备份**：
```bash
docker exec difyapp-postgres pg_dump -U postgres postgres > backup.sql
```

**RustFS 备份**：
```bash
# RustFS 数据存储在 Docker 卷中，可以直接备份卷
docker run --rm -v difyapp-dependencies_rustfs-data:/data -v $(pwd):/backup alpine tar czf /backup/rustfs-backup.tar.gz /data
```

**向量数据库备份**：
- Qdrant：备份 Docker 卷 `qdrant-data`
- Milvus：备份 Docker 卷 `milvus-data`
- FAISS：备份本地文件系统 `data/faiss` 目录

### Q3: 如何恢复数据？

**A**: 

**PostgreSQL 恢复**：
```bash
docker exec -i difyapp-postgres psql -U postgres postgres < backup.sql
```

### Q4: 服务启动失败怎么办？

**A**: 
1. 查看服务日志：`docker-compose -f docker-compose.dependencies.yml logs [service-name]`
2. 检查端口是否被占用
3. 检查资源是否充足（内存、磁盘空间）
4. 检查配置文件是否正确

### Q5: EasyOCR 首次启动很慢？

**A**: 首次启动时，EasyOCR 需要下载模型文件（约 1-2GB），可能需要 10-30 分钟。模型下载完成后，后续启动会更快。模型文件会保存在 Docker 卷中，不会重复下载。

### Q6: 如何查看服务资源使用情况？

**A**: 
```bash
# 查看所有容器的资源使用情况
docker stats

# 查看特定容器的资源使用情况
docker stats difyapp-postgres
```

### Q7: 如何停止所有服务？

**A**: 
```bash
docker-compose -f docker-compose.dependencies.yml down
```

### Q8: 如何删除所有数据？

**A**: 
```bash
# 停止并删除容器和数据卷
docker-compose -f docker-compose.dependencies.yml down -v

# 警告：此操作会删除所有数据，请谨慎操作！
```

### Q9: 如何更新服务版本？

**A**: 
```bash
# 拉取最新镜像
docker-compose -f docker-compose.dependencies.yml pull

# 重新创建容器
docker-compose -f docker-compose.dependencies.yml up -d
```

### Q10: 生产环境部署注意事项？

**A**: 
1. **修改默认密码**：所有服务的默认密码都必须修改
2. **配置数据备份**：定期备份所有数据
3. **配置监控告警**：监控服务健康状态
4. **配置资源限制**：合理设置 CPU 和内存限制
5. **配置网络安全**：使用防火墙和网络隔离
6. **配置日志收集**：集中收集和管理日志
