## 系统简介

DifyApp 是一个基于 Dify AI 平台的应用管理系统，为您提供便捷的 AI 应用使用体验。系统采用现代化的 RAG（检索增强生成）技术，支持多种向量数据库和 LLM 模型，提供完整的 AI 应用管理、知识库管理、智能问答等功能。系统支持：

- **智能应用**：使用管理员配置的 AI 应用进行对话和工作流处理
- **知识库管理**：创建和管理个人知识库，上传文档进行向量化
- **知识库问答**：基于知识库内容进行智能问答
- **对话历史**：查看和管理您的对话记录
- **AI 绘图**：使用自然语言生成各种类型的图表（流程图、架构图、思维导图等）
- **图片分析**：使用视觉模型分析图片内容（图片理解、文字识别、图表分析等）
- **实时信息检索**：通过浏览器检索功能获取实时网络信息

## 快速开始

### 1. 登录系统

1. 打开系统登录页面
2. 输入您的用户名和密码
3. 点击"登录"按钮

> **提示**：如果您还没有账号，请联系管理员为您创建账号。

### 2. 首次使用

登录成功后，系统会自动跳转到"智能应用"页面。您可以：

- 浏览可用的 AI 应用
- 选择应用开始使用
- 查看对话历史
- 管理个人知识库

## 功能模块

### 3.1 智能应用

智能应用页面展示了所有您有权限使用的 AI 应用。

#### 3.1.1 查看应用列表

- 应用以卡片形式展示，包含：
  - 应用图标
  - 应用名称
  - 应用描述
  - 应用类型（Chat Flow / Workflow）
  - 是否支持流式响应

#### 3.1.2 使用应用

1. 在应用列表中，找到您想要使用的应用
2. 点击应用卡片上的"使用"按钮
3. 系统会根据应用类型跳转到相应的交互界面：
   - **Chat Flow 应用**：进入聊天界面
   - **Workflow 应用**：进入工作流界面

#### 3.1.3 应用类型说明

- **Chat Flow**：对话式应用，支持多轮对话
- **Workflow**：工作流应用，支持复杂的任务处理流程

### 3.2 智能问答

智能问答功能允许您与 AI 进行对话交流。

#### 3.2.1 开始对话

1. 在导航栏中点击"智能问答"
2. 在页面中选择要使用的模型（如果未选择，使用默认模型）
3. 在输入框中输入您的问题
4. 如果需要上传图片，点击"上传图片"按钮选择图片文件（仅支持视觉模型）
5. 可以选择启用"浏览器检索"功能（获取实时网络信息）
6. 点击"发送"按钮或按 Enter 键发送消息

#### 3.2.2 对话功能

- **流式响应**：如果应用支持流式响应，答案会实时显示
- **问答/任务模式切换**：门户页输入区支持在“问答”和“任务”之间切换。问答模式用于普通对话；任务模式用于需要拆解步骤、调用工具或执行较复杂工作的任务。
- **Markdown 渲染**：支持 Markdown 格式的文本渲染
- **代码高亮**：代码块会自动进行语法高亮
- **数学公式**：支持 LaTeX 数学公式渲染
- **文件上传**：部分应用支持文件上传功能
- **图片输入**：如果使用支持视觉输入的模型，可以上传图片进行分析（图片理解、文字识别、图表分析等）
- **浏览器检索**：可以启用浏览器检索功能，获取实时网络信息

#### 3.2.3 任务模式

任务模式适合以下场景：

- 需要先拆解计划，再逐步完成的复杂问题。
- 需要调用 MCP 时间、位置、浏览器检索等工具辅助完成的任务。
- 需要读取可用 skills 作为能力上下文的任务。
- 可能涉及创建、修改、删除、备忘录等高风险动作，需要用户确认后继续执行。

**使用方式：**

1. 在门户页输入区选择“任务”模式。
2. 输入任务目标，例如“帮我调研并整理一个方案”。
3. 根据需要选择模型和是否启用浏览器检索。
4. 点击发送。
5. 页面会展示 Agent 执行过程，包括计划、步骤、工具调用、观察结果和最终答案。

**执行过程展示：**

- 顶部显示 Agent 当前状态，例如“Agent 正在执行”“等待确认”“执行完成”。
- 每个步骤以独立卡片展示，可单独展开或收起。
- 当前执行步骤会高亮显示。
- 工具调用和观察结果会归入对应步骤。
- 如果任务触发高风险动作，系统会显示确认卡片，用户可以选择“确认执行”或“拒绝”。
- 最终答案会展示在执行过程下方，收起执行过程不会隐藏最终答案。

**注意事项：**

- 任务模式第一版优先支持文本输入；图片附件仍建议使用问答模式或视觉模型能力。
- 系统不会展示模型隐藏推理链，只展示显式计划、动作、工具输入摘要、观察结果和结果摘要。
- 任务会话会与普通会话混合出现在最近会话和历史列表中，并显示“任务”标签。

#### 3.2.4 对话管理

- **新对话**：点击"新对话"按钮开始新的对话会话
- **继续对话**：系统会自动保存对话历史，您可以继续之前的对话
- **查看历史**：在"对话历史"页面查看所有对话记录

### 3.3 知识库管理

知识库管理功能允许您创建和管理个人知识库，上传文档并进行向量化处理。

#### 3.3.1 创建知识库

1. 在导航栏中点击"知识库管理"
2. 点击"创建知识库"按钮
3. 填写知识库信息：
   - **知识库名称**（必填）：为知识库起一个易于识别的名称
   - **描述**（可选）：简要描述知识库的用途和内容
   - **向量化模型**（可选）：选择用于文档向量化的模型，不选择则使用默认模型
   - **向量存储类型**（可选）：选择向量存储方式
     - **Qdrant**：分布式向量数据库（默认）
     - **FAISS**：本地文件存储
     - **Milvus**：分布式向量数据库
     - **Chroma**：轻量级向量数据库
     - **Weaviate**：开源向量数据库
     - **PgVector**：PostgreSQL扩展
     - **Elasticsearch**：分布式搜索和分析引擎
4. 点击"确定"创建知识库

> **注意**：知识库创建后，向量存储类型无法修改。

#### 3.3.2 查看知识库列表

知识库列表显示所有您有权限访问的知识库，包括：

- 知识库名称和描述
- 文档数量
- 可见性状态（公开/私有）
- 向量化模型
- 向量存储类型
- 创建时间

#### 3.3.3 编辑知识库

1. 在知识库列表中，找到要编辑的知识库
2. 点击"编辑"按钮
3. 修改知识库信息（注意：向量存储类型无法修改）
4. 点击"确定"保存修改

#### 3.3.4 删除知识库

1. 在知识库列表中，找到要删除的知识库
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除知识库将同时删除所有关联的文档和向量数据，此操作不可恢复！

#### 3.3.5 上传文档

1. 在知识库列表中，找到要上传文档的知识库
2. 点击"文档管理"按钮
3. 在文档管理页面，点击"上传文档"按钮
4. 选择要上传的文件（支持 PDF、Word、TXT、Markdown 等格式）
5. 点击"确定"开始上传

**文档处理流程**：
1. 文档上传到对象存储
2. 根据文档类型选择处理方式：
   - 图片文件和PDF文件：优先使用OCR服务识别文字
   - Word文档：提取图片和表格进行OCR识别，与文本内容合并
   - 其他格式：使用文档解析工具提取文本
3. **智能分块处理**：系统根据文件类型和内容特征自动选择合适的分块策略
   - 代码文件（.java, .js, .py等）→ 代码分块
   - 表格文件（.xls, .xlsx, .csv）→ 表格分块
   - Markdown文件（.md）→ 标题/段落分块（自动检测表格和代码）
   - Word文档（.doc, .docx）→ 段落分块（自动检测表格）
   - PDF文档 → 段落分块（自动检测表格）
   - 纯文本文件（.txt）→ 段落分块
   - 混合内容（表格+文本+代码）→ 混合分块策略
4. 使用向量化模型将文档块转换为向量
5. 向量数据存储到向量数据库中

**智能分块提示**：
- 文档上传成功后，系统会自动检测并显示使用的分块策略
- 例如："智能分块提醒：段落分块（自动检测表格）"
- 无需手动配置，系统会根据文件特征自动选择最优分块方式

**文档状态**：
- **pending**：等待处理
- **processing**：正在处理中
- **completed**：处理完成
- **failed**：处理失败

#### 3.3.6 文档管理

在文档管理页面，您可以：

- **查看文档列表**：查看知识库中的所有文档
- **下载文档**：下载原始文档文件
- **删除文档**：删除不需要的文档
- **重新向量化**：如果文档向量化失败，可以重新进行向量化处理

#### 3.3.7 文档格式支持

系统支持以下文档格式：

- PDF（.pdf）- 支持OCR文字识别
- Word（.doc, .docx）- 支持文本提取和图片OCR识别
- 文本文件（.txt）
- Markdown（.md）
- 图片文件（.png, .jpg, .jpeg, .gif, .bmp等）- 支持OCR文字识别
- Excel（.xls, .xlsx）
- PowerPoint（.ppt, .pptx）
- HTML（.html）
- 其他格式（根据系统配置）

**OCR识别说明**：
- 图片文件和PDF文件会自动使用OCR服务识别文字
- Word文档中的图片也会进行OCR识别，并与文本内容合并
- 如果OCR服务不可用，系统会自动回退到文本解析

### 3.4 知识库问答

知识库问答功能允许您基于知识库内容进行智能问答。

#### 3.4.1 开始问答

1. 在导航栏中点击"知识库问答"
2. 选择要使用的知识库
3. 选择问答模型（可选，不选择则使用默认模型）
4. 在输入框中输入您的问题
5. 点击"发送"按钮或按 Enter 键发送

#### 3.4.2 问答功能

- **流式响应**：答案会实时显示，提供更好的交互体验
- **非流式响应**：一次性返回完整答案
- **连续对话**：支持多轮对话，系统会记住上下文
- **答案来源**：显示答案引用的文档片段

#### 3.4.3 模型选择

- 系统会显示可用的问答模型列表
- 每个模型会显示：
  - 模型名称
  - 模型标识（带颜色区分）
  - 是否启用
- 选择模型后，系统会使用该模型进行问答

> **提示**：如果知识库的向量化模型被禁用，系统会显示警告提示。

#### 3.4.4 上下文压缩

在连续对话中，如果历史对话内容过长，系统会自动进行上下文压缩：

- **滑动窗口策略**：只保留最近的对话轮次
- **总结策略**：使用 AI 总结历史对话
- **混合策略**：结合滑动窗口和总结

### 3.5 对话历史

对话历史功能允许您查看和管理所有对话记录。

#### 3.5.1 查看对话历史

1. 在导航栏中点击"会话历史"
2. 系统会显示所有对话会话的列表
3. 每个会话卡片显示：
   - 会话标题
   - 会话类型（普通聊天/知识库问答/文档问答/任务）
   - 任务会话会显示“任务”标签

### 3.6 记忆管理

记忆管理功能允许您查看和管理个人记忆信息，系统会自动从对话中抽取有用的信息进行保存，并在后续对话中使用这些信息提供更个性化的回答。

#### 3.6.1 查看个人记忆

1. 在智能问答页面，点击右上角的"记忆管理"按钮
2. 系统会弹出记忆管理对话框
3. 对话框显示：
   - 记忆列表：显示所有保存的记忆信息
   - 记忆类型：长期记忆（文本）和实体记忆（结构化数据）
   - 记忆内容：显示具体的记忆信息
   - 更新时间：显示记忆最后更新时间
   - 作用域：显示记忆所属的作用域（聊天、知识库、应用）

#### 3.6.2 搜索和筛选记忆

**搜索记忆**：
1. 在记忆管理对话框的搜索框中输入关键词
2. 系统会实时筛选包含关键词的记忆

**筛选记忆**：
1. 使用类型筛选：选择"长期记忆"或"实体记忆"
2. 使用作用域筛选：选择"长期记忆"或"实体记忆"
2. 使用作用域筛选：选择"聊天"、"知识库"或"应用"
3. 可以组合使用多个筛选条件

#### 3.6.3 重置查询条件

如果您想重置所有的搜索和筛选条件：
1. 点击"重置查询条件"按钮（橙色警告按钮）
2. 系统会重置：
   - 搜索关键词
   - 记忆类型选择
   - 作用域选择
   - 分页位置
3. 显示"查询条件已清空"提示

#### 3.6.4 清空个人记忆

如果您想删除所有个人记忆信息：
1. 点击"清空记忆"按钮（红色危险按钮）
2. 系统会弹出确认对话框
3. 确认后，系统会：
   - 根据当前选择的筛选条件清空对应记忆
   - 如果未选择筛选条件，则清空所有个人记忆
   - 显示"记忆已清空"提示

> **注意**：清空记忆操作不可恢复，请谨慎操作！

#### 3.6.5 记忆类型说明

**长期记忆（Long-term Memory）**：
- 保存用户的偏好、背景信息、常用技术栈等
- 以文本形式存储，通常是一句话描述
- 例如："用户常用技术栈是Spring Boot + Vue 3"

**实体记忆（Entity Memory）**：
- 保存结构化的实体信息
- 以JSON格式存储，包含实体类型、名称和属性
- 例如：项目信息、组织架构、产品详情等

#### 3.6.6 记忆作用域

**聊天作用域（Chat）**：
- 在智能问答中产生的记忆
- 与用户的一般对话相关

**知识库作用域（Knowledge Base）**：
- 在知识库问答中产生的记忆
- 与特定的知识库内容相关

**应用作用域（App）**：
- 在特定AI应用中产生的记忆
- 与特定的应用场景相关

#### 3.6.7 记忆使用原理

系统会自动：
1. **抽取记忆**：从对话中识别和抽取有用的信息
2. **存储记忆**：将抽取的信息分类存储为长期记忆或实体记忆
3. **使用记忆**：在后续对话中，根据当前问题匹配相关的记忆信息
4. **更新记忆**：当检测到相关信息发生变化时，自动更新现有记忆

#### 3.5.2 搜索和筛选

- **搜索**：在搜索框中输入关键词，搜索会话标题或内容
- **筛选**：按会话类型筛选（普通聊天/知识库问答）
- **排序**：按时间排序（最新/最旧）

#### 3.5.3 查看会话详情

1. 点击会话卡片
2. 系统会显示该会话的所有消息
3. 消息以对话形式展示：
   - 用户消息：显示在右侧
   - AI 回复：显示在左侧
   - 支持 Markdown 渲染、代码高亮、数学公式

#### 3.5.4 继续对话

1. 在会话详情页面，点击"继续对话"按钮
2. 系统会加载该会话的历史消息
3. 您可以继续之前的对话

#### 3.5.5 开启新对话

1. 在会话详情页面，点击"新对话"按钮
2. 系统会创建一个新的会话
3. 您可以开始新的对话

#### 3.5.6 删除会话

1. 在会话列表中，找到要删除的会话
2. 点击"删除"按钮
3. 确认删除操作

> **警告**：删除会话将同时删除所有关联的消息，此操作不可恢复！

#### 3.5.7 更新会话标题

1. 在会话详情页面，点击"编辑标题"按钮
2. 输入新的会话标题
3. 点击"确定"保存修改

#### 3.5.8 导出会话

1. 在会话详情页面，点击"导出"按钮
2. 系统会生成包含所有消息的导出文件
3. 您可以保存或分享导出的会话内容

### 3.6 AI 绘图

AI 绘图功能允许您使用自然语言生成各种类型的图表，包括流程图、架构图、思维导图、时序图等。

#### 3.6.1 开始绘图

1. 在导航栏中点击"AI 绘图"
2. 选择图表类型（可选，不选择则使用默认类型）
3. 在输入框中输入您的绘图需求（自然语言描述）
4. 点击"生成"按钮或按 Enter 键

#### 3.6.2 支持的图表类型

系统支持以下图表类型：

- **流程图（flowchart）**：用于描述业务流程、决策流程等
- **架构图（architecture）**：用于描述系统架构、技术架构等
- **思维导图（mindmap）**：用于组织思维、知识结构等
- **时序图（sequence）**：用于描述系统交互序列
- **UML 图（uml）**：用于描述类图、用例图等
- **组织架构图（org）**：用于描述组织架构
- **网络图（network）**：用于描述网络拓扑
- **自定义图表（custom）**：根据需求自定义图表类型

#### 3.6.3 图表生成

## 常见问题

### Q1: 为什么我看不到某些应用？

**A**: 应用可见性由管理员控制。如果您看不到某个应用，可能是：
- 管理员未将该应用设置为对您可见
- 应用已被禁用
- 请联系管理员检查应用权限设置

### Q2: 文档上传后多久可以完成向量化？

**A**: 文档向量化时间取决于：
- 文档大小
- 文档格式
- 向量化模型性能
- 系统负载

通常，小文档（几页）在几分钟内完成，大文档可能需要更长时间。您可以在文档列表中查看处理状态。

### Q3: 为什么知识库问答的答案不准确？

**A**: 可能的原因：
- 知识库中缺少相关文档
- 文档向量化不完整
- 检索参数设置不当
- 问答模型选择不当

建议：
- 确保知识库中有相关的文档
- 检查文档是否已完成向量化
- 尝试使用不同的问答模型
- 联系管理员调整检索参数

### Q4: 如何修改密码？

**A**: 
1. 点击页面右上角的用户头像或用户名
2. 在下拉菜单中选择"修改密码"
3. 输入当前密码和新密码
4. 确认新密码
5. 点击"确定"完成修改

### Q5: 对话历史会保存多久？

**A**: 对话历史会永久保存，除非您手动删除。系统不会自动删除历史记录。

### Q6: 支持哪些文件格式上传？

**A**: 系统支持以下格式：
- PDF（.pdf）- 支持OCR识别
- Word（.doc, .docx）- 支持文本提取和图片OCR识别
- 文本文件（.txt）
- Markdown（.md）
- 图片文件（.png, .jpg, .jpeg, .gif, .bmp等）- 支持OCR识别
- Excel（.xls, .xlsx）
- PowerPoint（.ppt, .pptx）
- HTML（.html）

**OCR识别说明**：
- 图片文件和PDF文件会自动使用OCR服务识别文字
- Word文档中的图片也会进行OCR识别
- 如果OCR服务不可用，系统会自动回退到文本解析

### Q7: 知识库的向量存储类型有什么区别？

**A**: 
- **Qdrant**：分布式向量数据库，适合生产环境，需要独立服务
- **FAISS**：本地文件存储，无需额外服务，适合开发测试环境
- **Milvus**：分布式向量数据库，适合生产环境，需要独立服务
- **Chroma**：轻量级向量数据库，适合中小规模应用
- **Weaviate**：开源向量数据库，支持多种向量化模型
- **PgVector**：PostgreSQL扩展，适合已有PostgreSQL环境的场景
- **Elasticsearch**：分布式搜索和分析引擎，支持向量搜索

选择建议：
- 开发测试环境：使用 FAISS
- 中小规模应用：使用 Chroma
- 生产环境：使用 Qdrant、Milvus 或 Weaviate
- 已有PostgreSQL环境：使用 PgVector
- 需要搜索功能：使用 Elasticsearch

### Q8: 如何选择向量化模型？

**A**: 
- 创建知识库时可以选择向量化模型
- 不选择则使用系统默认模型
- 不同模型有不同的性能和成本，请根据需求选择

### Q9: 知识库问答支持连续对话吗？

**A**: 是的，知识库问答支持连续对话。系统会自动管理对话上下文，并在需要时进行上下文压缩。

### Q10: 如何联系管理员？

**A**: 如果您遇到问题或需要帮助，请联系系统管理员。管理员可以：
- 为您分配应用和知识库权限
- 帮助您解决技术问题
- 调整系统配置

### Q11: AI 绘图生成的图表不准确怎么办？

**A**: 可能的原因：
- 提示词描述不够清晰
- 图表类型选择不当
- 模型理解有偏差

建议：
- 使用更详细和清晰的提示词
- 尝试不同的图表类型
- 使用修改功能逐步调整图表
- 尝试使用快速模板作为起点

### Q12: 如何保存和分享图表？

**A**: 
1. 生成图表后，点击"保存"按钮
2. 输入图表名称并保存
3. 保存的图表可以在"我的图表"列表中查看
4. 可以导出为图片或复制代码分享

### Q13: 支持哪些图表类型？

**A**: 系统支持以下图表类型：
- 流程图（flowchart）
- 架构图（architecture）
- 思维导图（mindmap）
- 时序图（sequence）
- UML 图（uml）
- 组织架构图（org）
- 网络图（network）
- 自定义图表（custom）

### Q14: 如何上传图片进行分析？

**A**: 
1. 在智能问答页面，选择支持视觉输入的模型（如 Qwen-VL、GPT-4 Vision 等）
2. 点击"上传图片"按钮，选择图片文件
3. 输入问题（可选，如果未输入问题，系统会使用默认问题）
4. 点击"发送"按钮
5. 系统会使用视觉模型分析图片内容并回答您的问题

**注意事项**：
- 只有支持视觉输入的模型才能处理图片
- 支持常见图片格式：PNG、JPEG、JPG、GIF、BMP等
- 如果模型不支持视觉输入，系统会提示您选择支持视觉输入的模型

### Q15: 什么是浏览器检索功能？

**A**: 浏览器检索功能可以让AI获取实时的网络信息来回答您的问题。启用后，系统会：
1. 自动搜索与您问题相关的网络信息
2. 将搜索结果整合到对话上下文中
3. 基于实时信息生成更准确的答案

**使用场景**：
- 查询最新资讯
- 获取实时数据
- 了解当前事件
- 需要时效性信息的问答
