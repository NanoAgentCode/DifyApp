# DifyApp 系统文档

## 项目概述
- 基于 Spring Boot 与 Vue 3 的企业级 AI 平台
- 支持 AI 应用全生命周期管理
- 提供 RAG、Text2SQL、AI 绘图等核心功能
- 采用前后端分离架构，支持多租户与流式响应

## 项目特点
- 企业级架构：Spring Boot 3.x + Vue 3
- 多租户支持：细粒度权限控制
- RAG 技术：基于 LangChain4j 实现
- 流式响应：支持实时交互体验
- 多模态支持：文本、图片输入 + OCR
- 可扩展性：模块化设计，易于扩展
- 跨平台：支持 Web 与桌面应用（Tauri）
- 丰富集成：MCP 协议、多种向量数据库、LLM 模型

## 核心功能
### 用户认证与授权
- JWT 令牌认证机制
- 注册、登录、登出功能
- 密码修改、重置、找回
- 多租户与权限控制
- 用户与资源可见性管理
- 管理员审核机制

### 智能对话
- 支持 Chat Flow 与 Workflow 模式
- 多模型集成（Dify API）
- 流式与非流式响应
- 上下文理解与多轮对话
- 对话历史完整记录
- 会话管理与实时交互

### 知识库管理
- 创建、编辑、删除、查询知识库
- 支持 PDF、Word、Excel、TXT、Markdown 等格式
- 自动解析与文档分块
- 文档向量化与语义索引
- 多种向量数据库支持
- 版本管理与批量上传

### RAG 问答
- 基于知识库的智能问答
- 向量相似度搜索与语义检索
- 上下文增强生成
- 引用来源与可追溯性
- 可配置检索参数（Top K、阈值）
- 多知识库联合检索

### AI 应用管理
- 应用创建、编辑、删除、查看
- 配置管理（模型、参数）
- 可见性控制（公开/私有）
- 使用统计与分析
- 应用模板与快速创建

### Text2SQL 功能
- 自然语言转 SQL 查询
- 支持 PostgreSQL、MySQL、Oracle 等
- 数据源连接与管理
- SQL 结果可视化
- 查询历史记录

### AI 绘图功能
- 基于 Mermaid 的图表生成
- 支持流程图、时序图、类图
- 自然语言描述生成代码
- 图表预览与导出

### 系统管理
- 全局配置管理
- 数据源与模型管理
- 向量数据库配置
- Prompt 模板管理
- 用户管理（审核、禁用）
- 监控与日志管理

### MCP 协议支持
- 浏览器搜索服务（实时网络搜索）
- 地理位置服务（获取位置）
- 时间服务（当前时间、时区）
- 实时信息检测与更新
- 可扩展服务集成

### OCR 服务集成
- 图片文字识别
- 支持多种图片格式
- 批量处理
- 识别结果结构化输出

## 系统架构
- 前端层：Vue 3，运行于 3000 端口
- 后端层：Spring Boot，运行于 9090 端口
- 数据存储层：
  - 关系型数据库（PostgreSQL/MySQL）
  - Redis（缓存）
  - 向量数据库（Chroma/Qdrant/Milvus 等）
  - MinIO（对象存储）
- 外部服务：Dify API（LLM 服务）

## 技术栈
### 后端技术栈
- 核心框架：Spring Boot 3.5.8
- 编程语言：Java 17
- 构建工具：Maven 3.6+
- ORM：Spring Data JPA / Hibernate
- 数据库：PostgreSQL/MySQL/Oracle/MongoDB/Neo4j
- 向量数据库：Chroma/FAISS/Milvus/Qdrant/Weaviate/Elasticsearch
- 存储与缓存：MinIO / Redis
- AI 框架：LangChain4j 0.34.0
- 文档解析：Apache Tika / Apache POI
- 安全：JWT
- API 文档：SpringDoc OpenAPI 2.3.0

### 前端技术栈
- 核心框架：Vue 3.3.4
- 构建工具：Vite 5.0.8
- 状态管理：Pinia 2.1.7
- 路由：Vue Router 4.2.5
- UI 框架：Element Plus 2.4.2
- HTTP 客户端：Axios 1.6.2
- Markdown 渲染：marked, highlight.js, katex, mermaid
- 桌面应用：Tauri 2.9.6（可选）

## 快速开始
### 环境要求
- 后端：JDK 17+，Maven 3.6+，PostgreSQL/MySQL，Redis，MinIO，向量数据库
- 前端：Node.js 16+，npm/yarn，Git
- Tauri：Rust 1.70+（可选）

### 项目启动流程
1. 克隆项目：`git clone https://github.com/Yarao-Liu/DifyApp.git`
2. 后端配置：修改 `application.yml`，初始化数据库
3. 后端构建：`mvn clean install`，运行 `mvn spring-boot:run`
4. 前端安装：`npm install`，配置 API 地址
5. 前端启动：`npm run dev`
6. 访问系统：前端 `http://localhost:3000`，API `http://localhost:9090`

## 系统数据流
- 用户访问前端 → 登录请求 → 认证服务验证 → JWT 返回
- 创建知识库 → 上传文档 → 保存元数据 → 向量化存储
- 发起对话 → 请求至后端 → 检索知识库 → 调用 LLM → 返回结果
- 对话历史、日志记录完整留痕

## 功能模块关系
- 用户层：用户认证模块
- 应用层：智能对话、知识库、AI 应用、系统管理
- 服务层：Text2SQL、MCP、OCR
- 模块间依赖清晰，认证服务为入口，其余模块协同工作

## 功能模块
### 1. 用户认证模块
- 后端：注册、登录、JWT、密码管理、权限控制、用户可见性
- 前端：登录/注册页、密码操作、令牌管理、用户信息展示

### 2. 智能对话模块
- 后端：对话接口、流式响应、MCP/OCR 集成、上下文管理
- 前端：实时聊天界面、模式切换、历史管理、Markdown 渲染

### 3. 知识库模块
- 后端：文档上传、解析、分块、向量化、RAG 问答、版本管理
- 前端：上传界面、处理状态、问答界面、统计展示

### 4. AI 应用管理模块
- 后端：应用 CRUD、配置管理、可见性、统计、模板
- 前端：应用列表、配置界面、统计图表、详情查看
