# DifyApp 系统文档

## 项目概述
- 基于 Spring Boot + Vue 3 的企业级 AI 平台
- 支持 AI 应用全生命周期管理
- 实现现代化 RAG 架构
- 多租户、跨平台、可扩展

## 项目特点
### 企业级架构
- Spring Boot 3.x + Vue 3
- 清晰代码结构，遵循最佳实践

### 多租户支持
- 多用户、多租户场景
- 细粒度权限控制

### RAG 技术
- 基于 LangChain4j 实现
- 支持多种向量数据库

### 流式响应
- 支持流式与非流式响应
- 提升用户体验

### 多模态支持
- 支持文本、图片输入
- 集成 OCR 服务

### 可扩展性
- 模块化设计
- 易于新增功能

### 跨平台
- 支持 Web 和桌面应用（Tauri）

### 丰富集成
- 支持 MCP 协议
- 多种向量数据库与 LLM 模型

## 核心功能

- JWT 令牌认证

- 关系型数据库（PostgreSQL/MySQL）

## 技术栈
### 后端技术栈
- Spring Boot 3.5.8，Java 17
- Maven 构建，JPA/Hibernate ORM
- 多数据库支持（PostgreSQL/MySQL/Oracle/MongoDB/Neo4j）

- Tauri 支持桌面应用

## 快速开始
### 环境要求
- JDK 17+，Maven 3.6+
- PostgreSQL/MySQL 12+/5.7+
- Redis 6.0+（可选）
- MinIO（对象存储）
- 向量数据库（Chroma/Qdrant/Milvus 等）

### 1. 克隆项目
```bash
git clone https://github.com/Yarao-Liu/DifyApp.git
cd DifyApp
```

### 2. 后端启动
- 配置数据库连接（`application.yml`）
- 配置 Redis、MinIO、向量数据库
- 执行初始化 SQL 脚本
- 构建并运行：`mvn clean install` + `mvn spring-boot:run`

### 3. 前端启动
- 安装依赖：`npm install`
- 配置 API 地址（`api.js`）
- 启动开发服务：`npm run dev`

### 4. 访问系统
- 前端：http://localhost:3000
- 后端 API：http://localhost:9090
- API 文档：http://localhost:9090/swagger-ui.html

## 系统数据流
- 用户访问 → 前端登录 → 认证服务验证 → JWT 返回
- 知识库上传 → 文档解析 → 向量化 → 存储至向量库
- 用户提问 → 对话服务 → 检索知识 → 调用 LLM → 返回结果
- 对话历史记录持久化至数据库

## 功能模块关系
- 用户认证模块连接所有应用层
- 智能对话模块依赖知识库、MCP、OCR
- 知识库与系统管理模块联动
- AI 应用管理与系统管理协同
- Text2SQL 由系统管理模块提供支持

## 功能模块详情
### 1. 用户认证模块
- 后端：注册、登录、JWT、权限控制、用户管理
- 前端：登录页、密码管理、令牌自动管理

### 2. 智能对话模块
- 后端：多模式对话、MCP 协议、OCR 集成、上下文管理
- 前端：实时聊天界面、模式切换、Markdown 渲染、会话管理

### 3. 知识库模块
- 后端：文档上传、解析、分块、向量化、RAG 问答
- 前端：上传界面、文档管理、问答界面、处理状态展示

### 4. AI 应用管理模块
- 后端：应用创建、配置、可见性、统计、模板管理
- 前端：应用列表、配置界面、使用统计图表

### 5. 系统管理模块
- 后端：系统配置、数据源、模型、向量库、Prompt 模板、Text2SQL、用户管理
- 前端：配置界面、数据源测试、模型配置、用户审核

### 6. 其他功能
- 主题切换（深色/浅色）
- Markdown 渲染（代码高亮、公式、图表）
- 帮助文档与响应式布局
- 桌面应用支持（Tauri）
- API 缓存与统一错误处理
- 国际化预留接口

## 部署说明
### 后端部署
- 打包：`mvn clean package`
- 运行：`java -jar target/backend-*.jar`
- 生产建议：HTTPS、安全 JWT、日志与监控配置

### 前端部署
- 构建：`npm run build`
- 部署：将 `dist/` 部署至 Nginx/Apache
- 桌面应用：`npm run tauri:build`
